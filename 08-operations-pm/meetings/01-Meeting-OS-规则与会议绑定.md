# 01 — Meeting OS（会议操作系统）：规则 ↔ 会议绑定

**用途**：回答 **「这些规则在哪场会里执行、谁负责卡点」** —— 避免 **文档很全、会里从不检查**；与 **`08/cadence/00`**（周月季时长）、**`01` §2.5**（违规成本）、**`02` §1**（三元组会议纪律）、**`_registry/规则绑定与执行责任.md`**（**未执行谁担责**）对齐。

**性质**：**`meetings/`** 子系统 SSOT；具体纪要仍用 **`README.md`** 命名规范。

---

## 1. 会议 → 必须检查项（默认绑定）

| 会议（可改**会名**，**不可改检查项**） | 必须检查 | **卡点 Owner**（可兼任，须书面） |
|------------------------------------|----------|-----------------------------------|
| **需求评审会**（PRD / REQ / Epic） | **§1.2 会议动作**（三步）；**附录 A：R1～R6**；**`spec` 锚点**；**`新需求-最小入口-模板` 复制区** | **产品负责人** **拒收权**（不合规 **不得**讨论方案）；**工程接口人** 对 R1/R2 技术可行性 **一票质疑** |
| **方案 / 跨模块评审会** | **`02`** **输入自 / 输出至 / Impact Map** ≥3 条可勾选 | **产品负责人** 或 **`21`** 指定仲裁人 |
| **周会**（经营短站会） | **`22`** 红项 / NS 子指标趋势；行动项 **Owner+日期** | **`22/05`** 表中的 KPI Owner |
| **决策会**（优先级 / 资源 / Kill） | **`21/01`** 决策栈、**`21/05`** Score、**`12/05`** 人天池对拍 | **`21`** 仲裁人（见 **`21/03`**） |
| **复盘会**（实验 / 试点 / 迭代） | **`99`** 证据与结论；**`20`** 入库项 | **实验台账 Owner**（**`15/06`**）+ **产品** |

**没有会议绑定 = 规则停留在纸面**；**新增会议类型** 须先 **补本表一行** 再发邀请。

---

## 1.2 需求评审会 — 会议动作（强执行 · 只做三步）

> **缺任一步 → 不讨论方案**（当场宣布 **「材料不全」** 散会或改 **补材料专场**）；**禁止**「先聊想法再补表」。

| Step | 检查项 | 标准 |
|------|--------|------|
| **1** | **最小入口节** | PRD 含 **`_registry/新需求-最小入口-模板.md`** **复制区**（或同表 **已填实** 粘贴于文首 **§ 新需求最小入口**） |
| **2** | **R1～R6 + `spec` 锚点** | **`01` 附录 A** 可勾选 / 已链且 **全勾**；**口头需求无表** = **未过本步** |
| **3** | **Impact + 是否上 `21`** | **输入自 / 输出至 / 至少 1 条 Impact**；跨模块须 **≥3 条** 且对齐 **`02`**；**是否需上 `21`** 已勾选并 **有理由** |

**与方案评审分工**：需求评审 **只做上表**；**三元组扩写、链上细节** 仍在 **方案 / 跨模块评审会** 按 **`02` §1** 收口 —— 但 **需求评审不得空白放行 Impact**。

## 1.1 反滥用：当场执行 + 记 `违规记录.md`

- **执行人**：上表 **「卡点 Owner」** 与 **会议主持人** **同时为责**；冲突时 **产品负责人** 对 **R1～R6 / Impact** 有一票 **暂停议程**。  
- **执行时间**：**当场** —— 材料不齐 → **宣布**本次议程结束或改为「补材料专场」，**不得**默许带缺陷通过。  
- **记录**：**24h 内** 在 **`_registry/违规记录.md`** 追加一行（含 **违规类型**、**处置**、**链接**）。**未记 = 视同门禁未执行**（`21` 季审可追问）。
