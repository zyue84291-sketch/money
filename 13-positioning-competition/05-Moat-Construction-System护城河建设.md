# 05 — Moat Construction System（产品护城河系统）

**用途**：回答 VC 必问 **「别人为什么不能复制你？」** —— 把优势从 **「文档写得好」** 变成 **「可枚举、可加深、可投入」的壁垒类型**；与 **`13/00`～`04`**（理性论证）、**`24`**（心智）、**`25`**（生态）、**`26`**（终局）**同真**，**不**互相替代。

**性质**：**`13-positioning-competition/`** 的 **护城河建设 SSOT**；**`04-护城河拆解`** 写 **类型与证据链**，本篇写 **建设系统与缺口自检**。

---

## 1. 护城河类型（必须逐类写清「有 / 无 / 计划」）

| 类型 | 你要写清 | 典型证据 / 加深动作 |
|------|----------|----------------------|
| **数据护城河** | 是否存在 **数据飞轮**（更多交易 → 更好模型/匹配 → 更高转化） | **`07`** 事件字典、**`15`** 实验结论是否 **沉淀为默认策略** |
| **网络效应** | 用户 / 供给是否 **互相增强**（同边或跨边） | **`10`** 机制、**`23`** 复制时 **密度阈值** |
| **供给壁垒** | B 端是否 **锁定**（切换成本、培训、声誉链上、独占区域） | **`02-stakeholders/02`**、**`25`** 合同与 handover |
| **技术壁垒** | 是否 **不可替代**（须诚实：多数 Web3 旅游 **非**强技术独占） | **`spec/01`**、**`spec/14`** 与竞品 **差异清单** |
| **合规壁垒** | 是否形成 **准入门槛**（牌照、审计轨迹、托管叙事与真值一致） | **`16`**、**`spec/08-*`** |

**防呆**：至少 **2 类** 须达到 **「12 个月内可验证加深」** 的投入计划，否则 **Moat 叙事降级** 为 **执行优势**（须在 **`11`** 披露为 **execution play** 而非 **structural moat**）。
