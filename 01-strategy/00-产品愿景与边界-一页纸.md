# 产品愿景与边界（一页纸）

**项目**：TravelTrust（TrustLayer 取向的全球旅行托管与信任协议）  
**产品一句话**：链上 **USDC Escrow** 保障旅行订单资金；链下 **撮合、行程、证据、评价** 与 **运营后台** 完整闭环。

## 愿景

为全球当地向导与旅行者提供 **可验证预订、托管支付、可审计争议与信誉锚点**，长期支持 **区域化增长与协议治理**（叙事与实现分阶段，以 `spec` 为准）。

## 硬边界（非目标）

| # | 非目标 | 依据 |
|---|--------|------|
| N1 | 平台 **不托管用户私钥**（默认路径） | `spec/01` §7、`spec/06` |
| N2 | **不把治理币 / 区域融资叙事** 与 **订单 USDC 支付** 混为同一话术 | `spec/82`、`spec/81` §1.2 |
| N3 | **不以 DB 或前端成功** 作为「已支付」真值 | `spec/01` §3、`spec/10` F4 |
| N4 | 不在未过 **`spec/15` 附录〇** 与法务门禁时宣称 **监管级零残留** | `spec/15` 附录〇说明段 |

## 成功标准（产品侧摘要）

- **主链 Happy Path** 与 **`confirm-completion` → 评分（若适用）→ `release`** 全渠道一致。  
- **六区 MVP**（见 `spec/17`）可演示、可验收。  
- **对外材料** 与 **`spec/08-4`**、**`82/83/84`** 数字一致且可 `check-governance-doc-linkage`。
