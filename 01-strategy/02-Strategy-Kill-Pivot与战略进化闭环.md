# 02 — Strategy Kill / Pivot 与战略进化闭环（Strategy Evolution Loop）

**用途**：回答 **「战略什么时候该被推翻、而不仅是被优化」** —— 防止 **KPI 局部最优、方向错误却越跑越快**；与 **`01/00`～`01`**（愿景与路线）、**`21`**（决策栈）、**`22`**（NS）、**`99`**（假设证伪）硬绑定。

**性质**：**`01-strategy/`** 子系统；触发后 **强制**进入书面 **Strategy Review**，**不是**日常周会能悄悄替代。

---

## 1. 触发条件（满足任一 → 必须立项 Review）

| ID | 条件（须在 **`22`** 与 **`09/02`** 有 **可复核定义**） | 典型证据 |
|----|--------------------------------------------------------|----------|
| **SK-1** | **连续 3 个日历月** North Star **无增长**（或低于事先写的 **floor**） | **`22/04`** + **`22/03`** 月会附件 |
| **SK-2** | **CAC > LTV**（或 **回收期** 超过董事会/当季 **`21/01`** 写死的红线） | **`09/02`** 单位经济表 |
| **SK-3** | **`99`** 验证 **否定** 某条 **核心假设**（例如「供给愿意 X」「用户愿付 Y」） | **`99`** 证据包 + **`03`** 假设日志 |
| **SK-4** | **合规 / 资金通道** 变化使 **当前 TAM 路径** 在法务书面意见下 **不可辩护** | **`16`** + 法务 |
| **SK-5** | **NS 连续 4 周** 环比下降（与 **`21/00` §5 K1** 同触发） | **`22/03`** 周趋势 + **`22/00`** |
| **SK-6** | **同一核心 KPI 叶**（**`22/04` §2.1**）**连续 4 周 Critical** | **`22/04`** 状态列周更 |

> **SK-1** 与 **`21/06` Incident** 区分：后者偏 **短期暴跌与战情**；本文件偏 **慢性方向错误** 与 **战略级 Kill**。**SK-5/SK-6** 与 **`21/00` §5 K1/K2** **同义触发**，**强制**走本节 **§2** 流水线。

---

## 2. 强制动作（Kill / Pivot 流水线）

1. **Strategy Review（强制）**  
   - 参与者：**CEO / 产品负责人 / 增长 / 财务或 FP&A / 法务（按需）**  
   - 产出：**Kill（停止主叙事线）** 或 **Pivot（新主假设）** 的 **一页纸决议** + 日期。

2. **重写 `01-strategy`**  
   - 更新 **`00-产品愿景与边界`** 的 **非目标** 与 **边界**；  
   - 更新 **`01-路线图与版本节奏`** 与 **`spec/07`** Wave **接口表**（不得口头改 Wave）。

3. **重排 `21` 决策栈**  
   - 书面补丁 **`21/01`**；**`21/05`** 权重与 **`12/05`** 人天池 **当季重算**。

4. **回写指标与实验**  
   - **`22/04`**：删除或降级 **与新战略无关** 的 KPI 叶；  
   - **`15/06`**：在跑实验 **逐条** 标 **继续 / 停 / 改假设**。

5. **对外与融资**  
   - **`11`**、**`05/00`**：叙事与 Data Room **同步**，避免 **内部已 Pivot、对外仍旧故事**。

## 3. 本质

| 只有执行飞轮 | 加上本闭环 |
|----------------|------------|
| 把选定的事 **做到极致** | 还能回答 **「这事还应不应该做」** |
| 易 **局部最优** | 显式 **Kill / Pivot** 门禁 |
