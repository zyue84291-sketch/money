# 15 — 融资风险预案（Red Flag Defense）

**用途**：Partner Meeting 与 IC 的核心是 **「被挑战时你怎么回答」**——预先写 **问题 → 事实 → 结论 → 剩余风险」**；与 `06` 合规骨架互补：**06 偏政策与矩阵，15 偏路演口述与证据包**。

## 1. 监管问题（Web3 / 证券 / 跨境）

**预备题示例**：Token 是否证券、**哪些司法辖区运营**、营销是否触达受限用户、Travel+RWA 是否触发 **支付/汇款/电子货币** 许可。

- [ ] 每个问题 **1 页以内** 简报（事实 + 律师结论摘要 + **不做的事**）  
- [ ] 对外 Q&A 与 **`spec`** 边界 **一致**  

---

## 2. Escrow 与资金托管合法性

**预备题**：托管方资质、用户资金与自有资金隔离、破产远程、争议解决。

- [ ] **产品流程图** + **资金流图** + **法律结构图** 三图一致  
- [ ] 已备 **「若监管变化」** 的 Plan B（不展开对外，内部有）  

---

## 3. Token 证券与披露风险

**预备题**：SAFT/预售历史、二级市场、**内幕与锁仓**、团队持币减持规则。

- [ ] **无未披露的销售或承诺**  
- [ ] 与 **`13`** 解锁与治理 **同一叙事**  

---

## 4. 商业模式质疑

**预备题**：为什么不是 OTA、为什么链上必须、**Take rate 是否可持续**、大客户集中。

- [ ] 每条质疑对应 **`03` 护城河** 或 **`02` 单位经济** 数据  
- [ ] **集中度** 与 `25`/`26` 披露一致  

---

## 5. 执行与冷启动风险

**预备题**：双边冷启动、地推成本、**全球化** 运营复杂度、关键人依赖。

- [ ] **试点证据**（`08`/`99`）可 **逐条支撑** 回答  
- [ ] **组织图 + 招聘计划** 与资金用途一致  

---

## 6. 增长与竞争风险

**预备题**：巨头抄袭、开源分叉、**链上拥堵与 UX**。

- [ ] **竞争矩阵** 已更新（季更）  
- [ ] **失败场景** 诚实的 1 段 + **缓解动作**（机构更信）  

---

## 使用方式（PM 建议）

- **每周** 把新出现的尖锐问题 **记入 Q&A log**（与 `12` DD 问题合并或双向链接）。  
- Partner Meeting 前 **只复习 Top 10**，避免背稿过度。
