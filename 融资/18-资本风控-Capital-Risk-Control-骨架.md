## 1. Runway 模型（现金与等价物）

**要写清**：**Opex / Burn** 基准情景、**收入抵扣**、**一次性支出**（审计、诉讼、安全事件）；**月更** 或 **周更**（临近红线时）。

- [ ] **最低现金红线**（几个月 runway）已董事会/创始人书面共识  
- [ ] **收入延迟 / 成本超预期** 的压力测试表（至少 3 档）  
- [ ] 与 **`17` 月报** 中的 runway 表述 **同源**  

---

## 2. Treasury 策略（法币 + 链上）

**要写清**：**多签**、**冷热钱包比例**、**稳定币 vs 法币**、**银行通道单点**、**RWA/业务收入入账路径**；**谁有权调拨**、**限额**、**双人复核**。

- [ ] **公司账与链上金库** 对账节奏（见 `14` Finance）  
- [ ] **单一托管方 / 单一 CEX** 集中度风险已登记缓解措施  
- [ ] **密钥与灾备** 与 **`07`** 技术 DD **一致**  

---

## 3. Token 波动风险（若持有或依赖 Token）

**要写清**：**资产负债表上是否持有 Token**、**薪酬/供应商是否 Token 计价**、**金库是否暴露于单边行情**。

- [ ] **对冲 / 多元化 / 稳定币化** 策略（在合规允许范围内）有 **书面原则**  
- [ ] **不对社区做价格承诺**；内部 **风险阈值** 与对外 **分离**  

---

## 4. 市场下行预案（Bear Market）

**要写清**：**招聘冻结线**、**市场费用砍线**、**产品 scope 收缩**、**融资窗口假设恶化** 时的 **默认动作序列**。

- [ ] 与 **`10`** 下一轮融资触发条件 **联立**（推迟则启动 B 计划）  
- [ ] **核心团队保留** vs **增长团队** 分层，避免「全砍或全留」无预案  

---

## 5. 融资失败预案（本轮或下一轮未 close）

**要写清**：**可接受的 bridge 来源**、**收入化路径加速**、**战略非稀释资金**、**最坏情况**（有序收缩/并购接触）。

- [ ] **员工与核心伙伴沟通** 话术与 **法律边界** 已准备（避免误导）  
- [ ] 与 **`12`** 中 Pass 原因分析 **闭环**（用于修正故事与对象）
