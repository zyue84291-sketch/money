## 1. Crisis SOP（分级与触发）

**要写清**：**等级定义**（L1–L4）、**战时指挥（Incident Commander）**、**法务/公关/工程联席**、**何时停更社交媒体**。

- [ ] **桌面演练** 每年 ≥1 次（含 **`38` runbook**）  
- [ ] **与 `41` 平行**：投资人通知顺序见 **`17` §4**（或合并附录）  

---

## 2. Public Statement（对外声明模板）

**要写清**：**事实 → 影响 → 行动 → 联系方式** 结构；**多语言** 与 **法务预审** 流程。

- [ ] **模板库**（黑客、资金冻结、监管问询）内部可检索  
- [ ] **禁止**：猜测原因、指责用户、未证实损失数字  

---

## 3. 社区舆情（Web3）

**要写清**：**Discord/Telegram/X** 的 **mods 权限**、**FAQ 冻结版**、**谣言响应时限**。

- [ ] **mods** 背景与 **账号安全**（防社工）已培训  
- [ ] **与 `21` Token 运营** 沟通边界（无价格承诺）  

---

## 4. 媒体策略

**要写清**：**指定发言人**、** embargo**、**「无可奉告」替代句**、**关系媒体名单**。

- [ ] **深度稿件** 与 **`01` 叙事** **同源**  

---

## 5. 黑客 / 资金事件处理（与工程联动）

**要写清**：**首条公告时间目标**、**链上追踪话术**、**用户赔付原则**（若适用）——**全部法务预审**。

- [ ] **`38` §5** 与 **`41`** **同一时间表字段**（T+0、T+4h…）
