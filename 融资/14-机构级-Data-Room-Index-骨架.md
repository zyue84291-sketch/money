## 建议目录树（机构默认心智模型）

> 实际文件夹名可用英文，对内说明中英对照；**每叶子节点** 应有：文件名、版本、Owner、**仓库内路径或 Off-repo 位置**。

### Legal

- [ ] 公司注册文件、Good standing  
- [ ] 股权结构表、股东协议 / SPA（脱敏版）  
- [ ] 重大合同清单（客户、供应商、IP 转让）  
- [ ] **Token 相关法律意见书**（若适用，Off-repo 常见）  
- [ ] 诉讼与争议声明（无则书面确认）  

### Finance

- [ ] 历史报表、管理报表口径说明  
- [ ] 财务模型（与 `02`、`11` 同源）  
- [ ] 银行对账单摘要或资金证明流程  
- [ ] **单位经济** 与收入确认政策（尤其 Web3 收入）  

### Product

- [ ] PRD 索引、路线图（与对外 Roadmap 一致）  
- [ ] 核心用户旅程、定价与 packaging  

### Tech

- [ ] 架构图、依赖清单、发版流程（对齐 `07`）  
- [ ] 监控与 SLO、事故响应（可选但加分）  

### Security

- [ ] 审计报告、漏洞赏金、渗透测试摘要  
- [ ] 密钥与多签管理说明（**不含**密钥本体）  

### Compliance

- [ ] 监管矩阵（对齐 `06`）  
- [ ] KYC/AML、用户地区政策（若适用）  
- [ ] Escrow / 资金托管相关法律与产品说明（**你方项目关键点**）  

### User Metrics

- [ ] 与 **`22/07`**、**`07-metrics`** 同源的指标定义 + 导出样例  
- [ ] **事件字典**（与 `spec/08-4` 可对拍）  

### Contracts

- [ ] 重大客户/伙伴合同摘要或 redacted 全文  
- [ ] 模板合同（NDA、试点协议）
