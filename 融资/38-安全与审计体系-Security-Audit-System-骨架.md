## 1. Smart Contract Audit（智能合约审计流程）

**要写清**：**版本冻结 → 审计范围 → 修复 → 复测 → 上链 tag**；**主网升级** 是否 **再审计阈值**。

- [ ] 每次主网升级 **变更登记** 与 **`07`** 发版记录 **一致**  
- [ ] **Critical 未修复** 不上主网（或书面风险接受）  

---

## 2. Bug Bounty

**要写清**：**平台**、**范围**、**赏金分级**、**SLA**、**与内部 triage 的接口**。

- [ ] **范围与 out-of-scope** 对外可读  
- [ ] **`20`** 中可链到最新 **program 链接**  

---

## 3. Key Management（密钥管理）

**要写清**：**生成、存储、轮换、离职回收**；**人 vs HSM**；**开发/预发/生产隔离**。

- [ ] **无单点个人** 持有生产密钥（或例外已书面批准）  
- [ ] **演练**（年度）已排期  

---

## 4. Multi-sig Governance（多签策略）

**要写清**：**阈值**、**签名人角色**、**紧急暂停**、**与时间锁（timelock）** 的组合。

- [ ] 与 **`13` 治理**、**`37` DAO 边界** **一致**  
- [ ] **签名人变更** 有 **书面流程 + 链上可验证**  

---

## 5. Incident Response（安全事件响应）

**要写清**：**检测 → 遏制 → 根除 → 恢复 → 复盘**；**内外部通知顺序**（用户、投资人、监管）；**与 `41` 联合演练**。

- [ ] **On-call** 与 **升级路径**（含 CEO/法务）已文档化  
- [ ] **表格top 场景**（私钥泄露、合约被利用、RPC 劫持）有 **runbook**
