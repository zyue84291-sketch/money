## 3. 读法建议（避免「只有 PPT」）

0. **先填 `45`（可点开）与 `46`（留存/需求）**，再谈 **`50`（时代叙事）** — 否则易被归为「AI 完整、世界空洞」。**开窗口前** 过 **[`59`](./59-ENTERPRISE-PRODUCTION-MATRIX.md)** 全表。  
1. 先过 **`09-材料门禁与核对清单`**，再发任何对外 PDF / Deck。  
2. **所有对外数字** 必须能指回 **`02`** 登记表 §0、**`06`** 与 VDR 中已归档的口径说明（见 **`09`** 勾表）；财务情景须与 **`48`** 模型版本号一致。  
3. **敏感原件** 不入 Git：在 **`14`** / **`49`** 或 VDR 元数据中标注 **Off-repo + Owner**。  
4. **材料 vs 打法**：Deck 与真值对齐后，用 **`10`** 定本轮策略、**`11`** 定对象、**`12`/`52`** 管进程；**`13`/`14`/`15`/`16`** 在 DD 与 Partner 阶段 **同步更新**。  
5. **关账后**：**`17`** 管披露节奏，**`18`** 管现金与资产安全；发 Token 时 **`21`** 与 **`16`** 分工为 **运营 vs 退出叙事**。  
6. **DD 深水区**：**`37`**、**`38`**、**`39`** 常与 **`06`/`07`** 并行被问；**`40`/`41`/`42`/`43`** 补全规模化与长寿命协议叙事。  
7. **结构性诚实**：融资窗口前过 **`22`**，再扩写 **`01`～`21`**。  
8. **单文件白皮书**：发 **`44`**（或导出 PDF）；正文 **不手写新数字**，只 **引用** **`02`** 与 VDR 已审计摘要。
