## 1. Investor Update（月报 / 双周报）

**要写清**：**频率**（月报常见）、**固定结构**（亮点 / 数字 / 风险 / 求助）、**分发列表**、**版本**（全体投资人 vs 单家 side letter）。

**建议模块**：Executive summary → KPI 表（与 `02` 同源）→ 产品/里程碑 → 团队 → 现金流/Runway 区间（与 `18` 一致）→ **明确 asks**（招聘、 intro、策略反馈）。

- [ ] **发送日** 固定（如每月第 5 个工作日），避免随机  
- [ ] 月报中的 **所有数字** 可指回 **`22/07`** 或 **`spec/08-4`**  
- [ ] **坏消息** 有固定通道（不只在月报末尾一行）  

---

## 2. Board Meeting 机制

**要写清**：**例会频率**、**议程模板**、**预读材料截止时间**、**决议事项 vs 通报事项**、**会议纪要 Owner** 与存档位置。

- [ ] **Board pack** 目录结构固定（财务 / 战略 / 合规 / 人事敏感分拆）  
- [ ] **利益冲突**（关联交易、自我交易）**单列议程**  
- [ ] 与 **股东协议 / 章程** 中的董事会权限 **一致**  

---

## 3. 投后 KPI 披露规则

**要写清**：**对外（投资人）** 看哪些指标、**定义**、**更新频率**；哪些 **仅董事会**、哪些 **永不对外**（例如未审计的细项）。

- [ ] **指标字典** 与对外的 **`07-metrics`** / 月报 **同一版本**  
- [ ] **同比/环比口径** 在脚注中锁定，避免「每次换算法」  

---

## 4. 投后沟通 SOP

**要写清**：**谁回复投资人微信/邮件**（CEO-only vs 授权 CFO/COO）、**SLA**（如 48h _ack_）、** escalations**（法律、公关危机）。

- [ ] **材料外发** 仍走 **`09` 门禁**（含转发 Deck、未公开数据）  
- [ ] **intro 请求** 的 **优先级规则**（与业务目标挂钩，避免全拒或全接）  

---

## 5. 信息权限分层（Need-to-know）

**建议分层示例**：Public teaser → 签约投资人 Update → Board → 核心管理层 → **Off-repo 法务原件**。

| 层级 | 典型内容 | 载体 |
|------|----------|------|
| L1 全体股权投资人 | 月报、股东大会材料摘要 | 邮件 / 门户 |
| L2 董事会 | Board pack、审计节选 | 加密盘 / VDR |
| L3 Lead / 带 DD 条款者 | 扩展 Data Room | `14` 子集 |
| L4 内部 | cap table 完整版、薪酬 | 不外发 |

- [ ] 与 **Term Sheet / SHA** 中的 **信息权、MFN** **逐条对照**  
- [ ] **离职员工** 与 **顾问** 的访问 **及时回收**
