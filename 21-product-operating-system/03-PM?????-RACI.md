# 03 — PM 决策权边界（RACI）

**用途**：写清 **PM 能决定什么、必须升级什么**；与 **`spec/`** 冲突时以契约为准。

---

## 1. RACI 摘要（可贴到 Notion / Confluence）

| 事项 | Product（负责人） | Engineering | Legal/Compliance | Growth | Design/UX |
|------|-------------------|---------------|-------------------|--------|-----------|
| **Epic 优先级与范围** | **A / R** | C | C | C | C |
| **链上/资金行为描述** | C | **R / A** | C | I | I |
| **对外数字与话术** | C | C | **A** | C | C |
| **增长实验上线** | **A** | C | **C（合规 gate）** | **R** | C |
| **城市运营 SOP 细节** | C | I | C | **R** | I |
| **发版宣称与检查项** | C | **R** | C | I | I |

**图例**：R = 执行，A = 最终批准，C = 咨询须到场，I = 知情。

---

## 2. PM 明确 **不** 拥有的权力

- 修改 **合约参数、API 真值、订单状态机** 而不走工程变更与 **`spec`** 更新。  
- 在 **未法务审** 的前提下定稿 **对外融资材料、监管敏感表述**。  
- 以 **「体验」** 为由移除 **合规必显、关键确认、审计所需日志**。

---

## 3. 升级路径（与 `20/03` 一致）

1. **职能负责人** → 2. **产品负责人** → 3. **CEO**（L4 战略）→ 4. **董事会 / 投资人沟通窗口**（仅叙事与重大风险）。
