# 干系人全景矩阵（权力 × 利益）

**方法**：经典 **Power / Interest** 网格，用于沟通频率与材料深度决策。

## 矩阵（摘要）

| 干系人 | 利益强度 | 权力/影响 | 沟通策略 |
|--------|----------|-----------|----------|
| 个人旅行者 | 高（体验、资金安全） | 中（口碑、退款） | 高频：产品内文案、帮助中心、透明 Escrow 状态 |
| 向导 / 服务商 | 高（收入、信誉） | 中 | 中频：准入、质押说明、争议规则可读版 |
| 机构投资人 / LP | 高（回报、叙事） | 高 | 低频深度：**Data Room**、法务签核材料；对齐 `82/83/84/08-4` |
| 区域与合作方 | 中高 | 中高 | 中频：区域条款与数据权限，对齐 `83`、`84` |
| 监管 / 法务 | 合规结果 | 极高（一票否决） | Gate 同步：`08-1`、`08-4`、LEGAL-SIGNOFF |
| 社区 / 媒体 | 舆论 | 中 | 口径包 + FAQ；对齐 `../product-manager/26-项目 FAQ` |
| 内部工程 / 运维 | 交付与稳定性 | 高（排期与架构） | 日周节奏：`07`、PR、Runbook、附录〇 |

**决策权（谁拍板 / 谁 veto / 谁必须被通知）**：**不**在本表展开 —— 见 **`08-决策权力边界-DACI-RAPID-RACI与通知矩阵.md`**；PRD / 发布单须能 **脚注** 到 **`08`** 对应决策类型行。

## 利益冲突裁决（C 端 vs 用户 vs 法务 / 资本）

**问题**：多干系人 **同时声称 P0** 时，**谁覆盖谁** 不能靠会上嗓门。

**默认裁决栈**（与 **`21-product-operating-system/01-决策优先级规则-增长风控收入UX.md`**、**`21/02-跨模块冲突解决与升级路径.md`** 一致；**冲突时以 `21` 书面当季为准**）：

| 优先级（高→低） | 典型主张方 | 说明 |
|------------------|------------|------|
| **1** | **法务 / 监管一票事项** | 未过 **`16`**、**`spec/08-*`** 书面边界 → **不得**为增长或资本叙事让路 |
| **2** | **资金安全与订单真源（用户/向导「钱与履约」）** | 对齐 **`spec/01`**、**`19`**；**优于** 纯体验优化或内部便利 |
| **3** | **C 端增长与转化（可逆实验）** | 须在 **合规+资金** 之下；**强依赖 L2/L3 证据**（见 **`03/00`**） |
| **4** | **资本侧时间表 / 叙事** | **不**自动覆盖 **1～2**；若冲突 → **升级 `21` L3/L4** + **`01/03` 战略决策日志** 留痕 |

**程序**：

1. **RACI 单 Owner**（**`21/03`**）在争议 **48h** 内拉出 **一页纸**：选项、风险、回滚。  
2. **书面结论** 进入 **`21/07`** 或 **`01/03`**（战略级取舍）；**客服/对外话术** 同步 **`19`**。  
3. **禁止**「先上线再对齐法务」作为默认路径；若灰度 → 仅走 **`01` SSOT 契约** 灰度条款 + **`_registry`**。
