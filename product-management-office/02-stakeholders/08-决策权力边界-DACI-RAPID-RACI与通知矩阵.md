# 08 — 决策权力边界（Decision Authority）：DACI / RAPID / RACI 升级版

**用途**：在已有 **干系人画像**（`00` 权力×利益）与 **会议绑定**（`08-operations-pm/meetings/01`）之上，书面回答 CFO / 董事会 / 工程负责人必问的 **三件事**：

1. **谁最终拍板（Decider / Approver）？**  
2. **谁有一票否决（Veto）？**  
3. **谁必须被通知（Inform / Must-notify）——在决策前还是决策后、几小时内？**

**性质**：**`02-stakeholders/`** 内 **决策权 SSOT**；与 **`21-product-operating-system/03-PM决策权边界-RACI.md`** 互补 —— **`21/03`** 偏 **「事项→RACI」**；本文偏 **「决策机制模板 + 干系人侧 veto/notify」**。**冲突时**：以 **`21`** 当季书面 + **`01-strategy/03` 战略决策日志** 为准，并 **回填** 本文矩阵 **版本脚注**。

---

## 1. 三套框架怎么选（不必三张表全贴）

| 框架 | 最适合的决策类型 | 一行定义 |
|------|-------------------|----------|
| **DACI** | 产品方向、路线图切片、跨团队优先级 | **D**river 推进 · **A**pprover 拍板 · **C**onsulted 必须征询 · **I**nformed 必须通知 |
| **RAPID** | 资金/合规/重大商业条款、不可逆承诺 | **R**ecommend · **A**gree（职能签字）· **P**erform 执行 · **I**nput 输入 · **D**ecide **唯一**决策人 |
| **RACI** | 日常交付、运营 SOP、文档 Owner | **R**esponsible 执行 · **A**ccountable **唯一**担责 · **C**onsulted · **I**nformed |

**默认组合（企业级推荐）**：

- **战略 / 预算 / 法务监管类** → **RAPID**（谁 **Decide** 只能有一个）+ **Inform 列表**（`§4`）。  
- **产品 / 技术方案 / 里程碑取舍** → **DACI** + 与 **`21/01`** 决策栈对拍。  
- **迭代任务、Runbook 步骤、会议产出归档** → **RACI**（与 **`21/03`**、`22/05` KPI Owner 对齐）。

---

## 2. 决策类型 × 权力（模板 — 公司首版须填实名人 / 角色）

> 下表 **Approver / Veto / Must-notify** 为 **逻辑角色**；**首版** 在表末 **脚注** 写清 **具体职位或姓名**，并在 **`_registry/Day1-Owner-范围登记.md`** 或 **`22/05`** 可追溯到人。

| 决策类型（示例） | 推荐框架 | **Approver（拍板）** | **Veto（一票否决）** | **Consult / Input（须书面留痕）** | **Must-notify（至少）** |
|------------------|----------|----------------------|----------------------|-------------------------------------|---------------------------|
| **未过监管/资金门禁不得上线** | RAPID | CEO 或授权 **Release Approver** | **法务/合规**（`05`、`spec/08-*`） | 工程、产品、运营、财务 | 全体 **卡点 Owner** + **投资人关系**（若已承诺窗口） |
| **NS / 北极星定义或变更** | DACI | CPO / CEO | **董事会**或章程约定方 | Data、Finance、增长、产品 | `22` 读者、`01-strategy` Owner |
| **单笔超阈投放 / 增长预算重分配** | RAPID | CFO + CEO（双签或书面授权） | CFO（超财务红线） | 增长、产品、法务 | Finance、**`07/07`** 维护 Owner |
| **链上参数 / 合约升级 / 多签变更** | RAPID | 技术委员会 + 章程约定 **Decide** | **安全负责人** + **法务**（对外叙事） | 工程、运维、`spec` Owner | 运维、客服、`19-risk-operations` |
| **用户资损 / 大规模客诉 / 监管问询响应** | RAPID | CEO 或 **Crisis Decider** | **法务**（对外口径） | PR、客服、产品、工程 | 见 **`19`**、**`06/02`** |
| **Kill 实验 / 全量发布 / 回滚执行** | DACI + RACI | 产品负责人 + Release Approver | **SRE/发布负责人**（技术回滚否决权：「现在不能切」） | Data、法务（若涉用户承诺） | 当周 **`meetings/01`** 绑定会议全体卡点 Owner |

**Veto 使用规则（避免「隐形否决」）**：

- **Veto 必须书面**（纪要 / 邮件 / 工单），含 **理由 + 替代条件 + 最长等待时间**。  
- **紧急通道**：仅 **`21/06` Incident** 或 **`06/02`** 定义的 **P0 资金** 场景可 **先断流后补书面**，**24h** 内补登 **`01/03` 或 `21/07`**。

## 3. 按干系人速查（与 `00`～`07` 文件对齐）

| 干系人文件 | 典型 **Consulted** | 典型 **Veto** | 典型 **Must-notify（Inform）** |
|------------|-------------------|----------------|----------------------------------|
| `01` 旅行者 | UX、客服、信任研究 | **不**对商业条款直接拍板；**集体诉讼风险** 时法务升级 | 重大改价、改 Escrow 规则、数据出境 |
| `02` 向导 / 服务商 | 供给、运营、法务（条款） | **法务**（合同模板）、**合规**（准入） | 费率/质押/争议规则变更 |
| `03` 投资人 / LP | IR、Finance | **董事会**（章程事项） | 下轮融资口径、**指标重定义**（见 **`07/06`**） |
| `04` 区域 / 合作方 | BD、法务、`83/84` Owner | **合规**（区域牌照类） | 区域独占/数据驻留变更 |
| `05` 监管 / 法务 | 外部顾问可选 | **`spec/08-*` 未闭合** → **全公司级门禁 Veto** | 任何对外披露、上币/治理叙事、存储跨境 |
| `06` 社区 / 媒体 | 市场、PR | **法务**（公开承诺） | 活动涉及奖励、**可执行承诺** |
| `07` 内部工程 / 运维 | 架构、SRE、安全 | **SRE**（稳定性红区发布）、**安全**（漏洞热修策略） | 发版、回滚、索引/对账异常 |

## 4. 通知矩阵（Must-notify SLA — 默认）

| 事件 | **须通知对象** | **建议 SLA** | 归档位置 |
|------|------------------|----------------|----------|
| **Approver 已定案** | 全体 **I**nformed 行 + **执行 R** | 定案后 **4h 工作时间内** | 纪要 + 工单 |
| **触发 Veto 或「有条件通过」** | Driver + Approver + 被否决方 Owner | **当场 + 24h 书面** | **`21/07`** 或 **`01/03`** |
| **生产回滚完成 / 资金断流** | CFO、客服负责人、`22` 值周 | **1h 内** | **`06/02`** 台账 + **`19`** |
| **指标口径变更（影响对外/融资）** | Finance、IR、`07/06` Owner | **同周** | **`product-manager/31`** + **`07/00`** 版本行 |
