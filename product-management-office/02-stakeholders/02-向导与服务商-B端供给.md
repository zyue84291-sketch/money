# 干系人：向导与服务商（B 端供给）

## 角色边界

- **向导（Guide）** 与 **商家 / 服务商（Shop UI · `provider` API）** 在协议上分层；**SSOT**：`spec/87`。  
- **准入**：链上身份质押池 + Registry 方案 B；`spec/01` §4、`spec/81`。  
- **可接单条件**：`registry.approved`、质押档位、未冻结等；见 `spec/01` §4。

## 核心诉求

- 订单与档期清晰、**收款路径可预期**（链上 release 与费率）。  
- 争议与证据规则透明；`spec/03`。

## 产品触点

- 申请向导 / DID：`spec/64`、`spec/04`。  
- 质押与 `/staking`：`spec/14` §1.1、`spec/06`。

## 关联 spec

- 向导申请与 DID：`../spec/64-申请向导-行业标准与DID检查清单.md`。

**Approver / Veto / Must-notify** 见 **`08-决策权力边界-DACI-RAPID-RACI与通知矩阵.md`** §3（`02` 行）：**费率、质押、争议规则** 变更须 **法务 + 运营** 进入 **Consult**；**合规** 可 **Veto** 准入与合同模板。
