# 07 — 网络效应飞轮（**机制 · 指标 · 与 Token 层叠**）

**用途**：回答 **「生态网络效应怎么转起来？」** —— 不止终局口号（**`26/04`**），而是 **可观测的飞轮** 与 **每周能推的杠杆**。与 **`13/04`**、**`15/00`** 指标族对齐。

---

## 1. 主飞轮（**信任履约 · 双边密度**）

```text
Guide ↑ → 体验/履约质量 ↑ → Traveler 转化与复购 ↑ → 数据（匹配/风控）↑ → 匹配效率 ↑ → Guide 收入预期 ↑ → Guide ↑
```

| **环节** | **代表指标（示例名）** | **责任文档** |
|----------|------------------------|----------------|
| Guide ↑ | 活跃 Guide 数、**首响**、完单率 | **`18/01`**、**`18/07` §2** |
| 体验 ↑ | NPS、**争议率**、CSAT | **`19/02`**、**`14/02`** |
| 用户 ↑ | MAU、**Commit→Escrow** | **`22/07`**、**`14/01`** |
| 数据 ↑ | 覆盖率、标签质量、埋点完整率 | **`07/00`**、**`15/04`** |
| 匹配 ↑ | `match_rate`、`time_to_match` | **`18/07` §2** |

**周会动作**：至少选一个 **环节** 定 **一个** 本周 owner + 目标 delta（链 **`22/06`**）。

---

## 2. 叠飞轮（**用户 · Token 使用 · 流动性 · 生态**）

> **合规前置**：本节 **不** 预设代币已发行；若 **无 Token**，则 **省略「Token 使用」环**，仅用 **Escrow + 信誉** 闭环。

```text
用户（与 GMV）↑ → Token 使用场景（若已上线）↑ → 链上流动性 / 深度 ↑ → 生态伙伴（钱包·DEX·链）意愿 ↑ → 用户获取效率 ↑
```

| **环节** | **风险 / 门禁** |
|----------|------------------|
| Token 使用 ↑ | **须** 满足 **`26/07`**「行为—是否用 Token」；否则 **叙事降级** |
| 流动性 ↑ | **成本** 记入 **`25/05`** Token 行；**不得** 隐藏做市支出 |
| 生态 ↑ | **集中度** 见 **`26/05`** |

---

## 3. 与 Stage 的耦合

- **Stage 1**：飞轮 **慢**，主 KPI 是 **集成稳定 + 成本**。  
- **Stage 2**：飞轮 **加速**，主 KPI 是 **归因 GMV + 争议率**。  
- **Stage 3**：飞轮 **可对外联合叙事**（仍须 **`05/00`** 门禁）。
