# 07 — Token 效用绑定（**行为 × 是否必须用 Token · 与产品核心关系**）

**用途**：回答 **「Token 为什么存在？」** —— 防止 **Token=仅融资工具** 叙事；与 **`26/02`** 协议路径、**`16`** 证券敏感区、**`spec/82` / `governance-token/`** **对齐**。本表为 **产品叙事 SSOT**；**上链行为** 以工程 **`spec/01`** 为准。

---

## 1. 行为绑定总表（**默认策略 · 须法务签注后对外**）

| **用户 / 协议行为** | **是否必须用 Token** | **说明** |
|---------------------|----------------------|----------|
| **浏览 / 搜索 / 收藏** | **否** | Web2 体验基线 |
| **预订（意向 / 下单）** | **否** | 法币与链下合同可完成 |
| **押金 / Escrow 托管（链上路径）** | **是（若产品路径选链上托管）** | **信任核心**；与 **`09`、`14`** 一致 |
| **争议仲裁 / 质押裁决（若机制如此设计）** | **是（若治理设计绑定）** | **降低合谋**、对齐 **`19`** 升级路径 |
| **质押成为 Guide / 信誉保证金** | **是（若采用链上信誉）** | **供给侧信任**；退出规则见 **`26/02`** |
| **普通支付路由（法币）** | **否** | **PSP** 主导；Token **不** 强绑刷卡 |
| **积分 / 返现（可选）** | **否（可用积分）** | 若 **双轨** 须披露 **兑换与稀释** |
| **治理投票（协议参数）** | **是（若发行治理 Token）** | 与 **`governance-token/`** 一致 |

**符号**：上表 **「是」** 表示 **产品设计允许/要求 Token 参与**；**不** 自动等于 **任何辖区合法** —— **`16/04`** 个案 gate。

---

## 2. 与 **`25/03`、`25/05`** 的接口

- **Web3 伙伴**：只承接 **表中「是」** 的行为层，避免 **钱包为了集成而硬发币**。  
- **单位经济**：Token 相关收入 **不稳定** 行（**`25/05`**）须与本表 **同一假设**。
