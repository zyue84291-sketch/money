# 黑天鹅与用户投诉升级

## 1. 升级级别（示例）

| 级别 | 触发 | 动作 |
|------|------|------|
| L1 | 单用户投诉 | 客服模板、`19/01`～`02` |
| L2 | 多用户同类问题 / 媒体询问 | 运营负责人 + PR + **`16`** |
| L3 | 资金类群体事件 / 监管问询 | **高管 + 法务 + 工程 on-call** + `Runbook` |

## 2. 与 **18** 联动

- 城市活动事故 → **本地负责人** 首响（`18/03` SOP）。

---

## 3. **每类用户话术（L2 / L3）**

| **用户类型** | **典型触发** | **对外口径（须 PR/法务预审后照读）** | **`ticket_type`** |
|--------------|--------------|----------------------------------------|-------------------|
| **Traveler / Guide** | 社交媒体 **@官方** 骂战 | 「我们已关注到相关反馈，**正在核查**，有结论后 **第一时间** 在 **官方渠道** 同步。」 | `social_escalation` |
| **渠道 / KOL / BD** | 要求 **撤合作 / 索赔** | 「请 **书面** 发送至 **partner@**（占位）；商务 **3 工作日** 内回复 **受理号**。」 | `partner_inquiry` |
| **媒体** | 采访提纲 | 「请发 **邮件至 press@**；**非经 PR** 一线 **不接受** 语音即兴采访。」 | `media_inquiry` |
| **监管 / 律师函** | 正式函件 | 「我们已收到并将由 **法务** 专项回复；**客服不解读法律条款**。」 | `regulatory_inquiry` |

**L1 日常投诉** 仍用 **`05` 话术大全**；**本节仅升级态**（详稿 **`05` §5**）。

---

## 4. **与 Runbook 分工（黑天鹅）**

| **工程** | **运营 / PR / `19`** |
|----------|----------------------|
| **启动 incident**、**技术 RCA**、**服务降级 / 迁移** | **用户广播模板**、**客服排队策略**、**舆情监测** |
| **链上紧急 pause**（若有） | **对外** 只发 **已批准** 短讯；**不**抢跑工程结论 |
| **数据导出给监管** | **法务** 主笔；运营 **配合工单索引** |

---

## 5. **内部工单字段（升级类 = 基线 + 扩展）**

**基线**：**`../20-product-iteration/00-用户反馈收集与闭环.md` §2**（**`priority` 至少 P1**）。

| **追加字段** | **必填** | **说明** |
|----------------|----------|----------|
| `escalation_level` | ✔ | `L2` / `L3` |
| `external_party` | ✔ | `media` / `regulator` / `lawyer` / `partner` / `other` |
| `pr_approved_text_url` | L2+ | 对外口径 **存档链接** |
| `legal_owner` | L3 | 法务 Owner user id |
| `incident_bridge_id` | 若有 | 与 **工程 incident** **对齐** |

---

## 6. **SLA（升级工单）**

| **级别** | **对「外部」首响** | **内部** |
|----------|-------------------|----------|
| **L2** | **≤2h**（值班负责人 **书面** 一条，可邮件） | **24h** 内 **PR + 运营** 联合纪要入工单 |
| **L3** | **监管/律师**：**法务 24h 内** 确认「收到」 | **工程 on-call** 按 **Runbook** **15 min** 内接单（与 **`22/02` §3** 指挥台联动） |
