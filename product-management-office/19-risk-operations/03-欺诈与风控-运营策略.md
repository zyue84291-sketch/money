# 欺诈与风控（运营策略）

## 1. 运营动作清单（与 `spec/03` 对齐）

| 风险 | 运营信号 | 动作 |
|------|----------|------|
| 刷单评价 | 短时间多评价同对 | 冻结评价展示、升级风控 |
| 女巫注册 | 同设备多号 | 与 rate limit、黑名单 |
| 证据 DoS | 超大附件 | 拒收并指引分块 `spec/01` |

## 2. Admin 与工具

- **`spec/70`**、**`spec/04` §3.4** Admin 路由 —— 运营 **不得** 越权改链上状态。

---

## 3. **与 Runbook 分工（欺诈 / 风控）**

| **工程（Runbook + Admin API）** | **运营（`19`）** |
|----------------------------------|------------------|
| **实施** 黑名单、rate limit、**关接口** | **开工单**、**对用户话术**、**冻结「展示层」**（评价隐藏等 **产品配置**） |
| **链上标记**（若合约支持） | **仅** 在 **法务书面** 后可配合 **提交链上操作申请**（双人复核） |
| **日志取证、关联设备指纹** | **不** 自称「我们已定罪」；统一 **「在调查中」** |

---

## 4. **每类用户话术（举报 / 被质询）**

| **用户类型** | **场景** | **L1 标准口径** | **`ticket_type`** |
|--------------|----------|-----------------|-------------------|
| **Traveler** | 举报向导刷单 / 威胁 | 「感谢您的举报，我们 **严肃对待**。请提供 **订单号** 与 **截图**；**24h 内** 给您 **是否立案** 的回复。」 | `fraud_report_traveler` |
| **Guide** | 举报旅行者恶意差评 / 逃单 | 「我们会 **双边核查**；在结论前 **评价展示** 可能暂时隐藏。请勿私下对骂。」 | `fraud_report_guide` |
| **任意** | 被要求「私下转账解决」 | 「**平台外付款** 不受保护；请 **仅在 App 内** 完成托管支付。」 | `policy_reminder`（**`08/`** 登记枚举） |

**详稿与禁区**：对齐 **`05`** 原则句 + **`01` 话术禁区**。

## 5. **内部工单字段（欺诈类 = 基线 + 扩展）**

**基线**：**`../20-product-iteration/00-用户反馈收集与闭环.md` §2**。

| **追加字段** | **必填** | **说明** |
|----------------|----------|----------|
| `reporter_role` | ✔ | `traveler` / `guide` / `observer` |
| `accused_party_id` | 若有 | 被举报方用户 id |
| `fraud_pattern` | ✔ | `review_ring` / `sybil` / `evidence_dos` / `other` |
| `freeze_actions_applied` | ✔ | 多选：`hide_reviews` / `limit_orders` / `none` |
| `risk_score_snapshot` | 若有 | 风控系统输出，**不得**展示给用户 |

## 6. **SLA（欺诈工单）**

| **级别** | **对举报人 L1** | **内部定案路径** |
|----------|-----------------|------------------|
| **P0**（人身威胁 / 大额疑似洗钱） | **≤10 min** | **2h** 内 **Risk + 法务** 联席备注入工单 |
| **P1** | **≤2h** | **3 工作日** 内初判 |
| **P2** | **≤24h** | **7 工作日** 内结案或升级 **`04`** |

**数字 SSOT**：与 **`../12-org-scaling/03-KPI体系-业务与组织.md` §5** 冲突时 **以 §5 为准**。
