# 04 — 路线 → Sprint：**任务粒度**（Epic → Feature → Story → Task）

**用途**：填 **「季度主题」与「双周 Sprint」之间的真空** —— 评审时工程能问：**接口改几条？几个 Story？谁估点？** 若只能答「本季度做信任系统优化」→ **会议当场卡住**。**工单系统 ID** 可与 Jira/Linear 字段 **同名映射**。

## 1. 层级定义（**写死用词**）

| 层级 | **定义（一句）** | **评审必须露出** |
|------|------------------|------------------|
| **Epic** | 季度内 **可交付的一坨价值**（常跨多 Sprint） | Epic 描述 + **NS / 漏斗指标** 绑定（**`22/04`** / **`15/00`**） |
| **Feature** | Epic 下 **可独立演示** 的用户能力切片 | **接口/合约/页面** 范围（链 **`spec/04`、`spec/13-1`**） |
| **Story** | Feature 下 **可验收** 的用户故事（INVEST） | **验收标准** + **埋点**（**`07/00`**） |
| **Task** | Story 拆到 **单人可完成** 的工程任务 | **前后端/数据/测试** 子任务 + **估点（人天或人周）** |

**反模式**：Epic 直接挂 **Task** 无 Story → **回溯与验收不可控**。

---

## 2. 样例（**Q2 主题：提升 Escrow 转化** — 可直接复制进评审表）

| 字段 | 内容 |
|------|------|
| **季度主题** | 提升 **Escrow** 转化（对齐 **`15/01` §2**、**`07/02`**） |
| **Epic** | **支付体验优化**（Epic ID 例：`EPIC-PAY-2026-Q2`） |
| **Feature** | **钱包连接优化**（`FEAT-WALLET-CONN-01`） |
| **Story** | **减少 1 步 `wallet_explain`**（`STORY-0142`，与实验 **`EXP-2026-0142`** 同源） |
| **Task（例）** | **前端**：改 CTA / 折叠步骤 UI（`TASK-FE-101`） |
| **Task（例）** | **后端**：埋点 `wallet_explain_*` 对齐 **`spec/04` §3.4**（`TASK-BE-204`） |
| **Task（例）** | **数据**：看板切片 + 分流校验（`TASK-DS-088`） |

**评审时工程可问**：「`TASK-BE-204` 动哪些 **路由**？是否 **`spec/04` Implemented**？」→ **须能指到行号或 OpenAPI tag**。

---

## 3. 与 Sprint 边界

- **一个 Sprint** 内 **承诺完成** 的应是 **Story 集合**（或明确 ** carry-over** 规则）；Epic **可跨 Sprint**，须在 **`03` 纪要** 标 **% 完成**。  
- **与 `spec/07` Wave**：Epic 须能在 **`../17-execution-roadmap/04-与工程07-Wave的接口表.md`** 找到 **Wave 行**。
