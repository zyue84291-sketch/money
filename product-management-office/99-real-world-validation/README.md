### 商业真实验证（无效验证定义）

任何 **`99`** 产出（试点、可用性、漏斗）**必须能回答**下面 **3 个问题**（与 **`09`**、**`22`**、**`09/06`** 同源口径），**否则视为无效验证**（不可单独支撑融资/发版/扩城宣称）：

1. **有没有带来真实 GMV**（或你们定义的 **有效订单量** 等价指标）？  
2. **有没有带来真实收入**（与 **`09/00`**、**`spec/08-4`** 一致口径）？  
3. **有没有改善现金流或单位经济**（回款、退款、占用、**burn** —— 见 **`09/06`**）？

与 **`../09-business-model/README.md`** **「现金真实性校验（Real Money Check）」** **三问**（真实收入 / 可回款 / 改善现金流）**同向**；**不满足**且 **无已登记风险承受** → **`09`** 所称 **虚增长**，**不得**单独支撑扩量或对外叙事。

> 若仅回答「用户喜欢 / NPS 高」而无 **上列三者之一**，只能作为 **定性输入**，须回 **`15`** 再设计 **可证伪实验**，**不得**替代 **商业闭环**。
