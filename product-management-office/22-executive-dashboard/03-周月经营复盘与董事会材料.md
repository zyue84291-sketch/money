# 03 — 周 / 月经营复盘与董事会材料

**用途**：把 **分析能力** 变成 **管理节奏**；与 **`15/04`**、**`07/01`** 互补（本文件偏 **exec 一页纸**）。**会议可打印版**见 **`06-周会模板-30min-会议执行版.md`**（**固定 30 min** 亦可嵌入 60 min 会的前半段）。

**与财务 / 风险节奏**：**LTV·CAC**（月）、**Risk 红项**（周）、**GMV / Revenue**（日或 T+1）的 **默认频率、Owner、会议落点** 见 **`09/05`** **§财务与风险更新节奏**；周会 **红项栏**、月复盘 **须核对** 是否 **按期发布**（与 **`22/05`** Owner 表一致）。

---

## 0. 决策触发规则（强制 · 与 **`21/00` §4** 同源）

**满足 `21/00` §4 表格 T1～T4 任一** → **当周必须** 产出 **`21` 书面结论**（选项 A/B、**`22/04` 叶子影响**、Owner）；**议程上** 须单列 **「进 `21`」**，**禁止**「只讨论无结论」散会。

**未触发 T1～T4** → **当周 exec 会** **不允许** 进入 **>15 min** 的 **开放式策略重排**（仅 **信息同步 + 执行对齐**）；若团队仍要讨论战略 → **先书面立项** 下周 **`21`** 联席。

---

## 0.1 决策责任绑定（周会执行 · 与 **`21/07` §5** 同源）

- **`21`** 结论宣读时 **必须** 点名 **决策 Owner** + **`22/04` 指标 Owner** + **`21/01` §4 执行 Owner**（可同人，须 **写全**）。  
- **`21/07` §4** 验证票 **到期** 当周：议程 **固定 5 min** 走 **责任链纠偏**（未写入 **`20/03`** = **会议未完成**）。

---

## 0.2 上周决策执行检查（强制 · **Execution Tracking**）

> **验证窗（`21/07` §4）** 管 **「对不对」**；本节管 **「做没做」** —— **更基础**。每周 **第一栏议程**（可与 **`06`** 会前 5 min 合并）过下表。

**主持人** 对 **上周及更早仍开放** 的每条 **`21` 决议**（含 **`21/01` §5** Task 行）逐项问 —— **Task 级状态** **必须** 与 **§5 表 `Status` 列** **同源**（**`Not Started` / `In Progress` / `Blocked` / `Done`**）：

| 决议 ID | Task # | **Status**（与 **`21/01` §5** 同列） | **是否达预期**（若验证窗到） | **是否需调整** |
|---------|--------|-------------------------------------|------------------------------|----------------|
| | | | Y / N / 尚早 | Y / N —— 若 Y → **当场改 Owner / 日期 / Status** |

- **Blocked** 须写 **阻塞物 + 新 Owner**；**不得** 静默跨周。  
- **Not Started** 且 **已逾拆解截止日期** → **升级 `20/03`** 或 **`21/02` L2**。  
- **Done** 但 **KPI 未改善** → **`22/03` §0.5**（或 **`21/01` §5.1**）—— **A/B/C** 必选。

---

## 0.3 决策容量限制（Decision Capacity · **防过载**）

| 规则 | 说明 |
|------|------|
| **每周新增 `21` 决议 ≤ 3** | 与 **§1「本周押注」Top-3** **对齐**；第 4 条须 **顺延** 至下周纪要 **或** **替换** 当周已 **书面撤回** 的决议槽 |
| **未完成 `21` 决议（含开放 Task）> 5** | **禁止** 当周再记 **新决议**（仅 **§0.2 纠 Blocked / Kill 结案**）；**例外**：**产品负责人** **签字置换** —— **关闭或降级 1 条** 最旧未完成决议后，**方可** 开 **1 条** 新槽 |

**主持人** 在周会 **Step「是否进 `21`」** 前先报数：**本周已用槽位 / 3**，**开放 backlog / 5**。

---

## 0.4 本周 Top-3 动态复核（强制 · **防静态优先级**）

> **Top-3 不是「写一次管一季」** —— **每周** 在定 **新** Top-3 之前，须先 **过堂上周书面 Top-3**。

**议程 §1 第 4 节「本周押注」首 5 min（强制）**：

1. **上周 Top-3 是否仍成立？** —— 逐条 **Y/N**；**N** 须 **一句理由**（**`22` 红项 / `21` 新触发 / Blocked / 外部**）。  
2. **是否需要替换？** —— 若 **≥1 条不成立** → **允许替换** 当周槽位；**替换** 须 **链 `22/04` 叶子** + **写进当周纪要**（**SSOT**）。  
3. **无复核即宣布新 Top-3** → **主持人** 可按 **`08/meetings/01` §1.1** **暂停** 本节，直至补 **§0.4** 勾选项。

---

## 0.5 执行失败处理（Failure Loop · **战术层**）

> **详版与选项定义**见 **`21/01` §5.1**；本节为 **周会必念** 的 **收口句**。

**当** **`21/01` §5** 中 **任一 Task = `Done`**，且 **`21/07` §4** 对应验证 **未达预期 / KPI 无改善** —— **当周周会** **必须** 产出 **A / B / C** 之一（**书面**，入 **`20/03`**）：

| 选项 | 一句话 |
|------|--------|
| **A** | **继续** —— **加资源** 或 **延长窗** |
| **B** | **调整** —— **改方案** + **新 Task**（**Status** 从 **Not Started** 重记） |
| **C** | **停止** —— **Kill** 该路径；释放 **§4** 资源；必要时 **`21/06`** |

**未选** → **不得** 新开 **同链路** 的 **放大实验**（**`15`**）。

## 1. 周会（60 min 模板 · 与 **`06`** 对齐）

> **前 30 min** 建议 **严格按** **`06-周会模板-30min-会议执行版.md`** 执行（**§0.2** → NS → Top3 KPI → **§0.3 容量** → **是否触发 `21`** → 若触发则 **选项/指标/结论 + `21/01` §5 拆解（含 Status）**）；**§0.4～§0.5** 在 **60 min** 会内 **必须** 出现。

0. **上周决策执行 + 容量**（**5 min，强制**）—— **§0.2** 表逐项（**更新 `21/01` §5 Status**）；报 **§0.3** 槽位（**/3**、**open≤5**）。  
1. **北极星 + 3 个支撑**（10 min）—— 仅 **`22/00`** 与 **`22/04`** / **`01`** 树顶 3 叶；**对照 §2.1 阈值与状态**。  
2. **红项**（15 min）—— 来自 **`02`**、**`22/04` Critical/Warning**、客服主管。  
3. **验证窗到期 + 执行失败**（**8 min**）—— **`21/07` §4** 到期票；**Done 且 KPI 未改善** → **§0.5** **A/B/C** 入 **`20/03`**。  
4. **本周押注**（20 min）—— **首 5 min：`§0.4` 复核上周 Top-3**；其后最多 **3 条** 战术动作，owner + 验收日 + **`22/04`** 叶子。  
5. **升级队列**（5 min）—— 进入 **`20/03`** 或 L4。
