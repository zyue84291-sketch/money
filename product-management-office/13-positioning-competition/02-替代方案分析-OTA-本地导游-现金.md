# 替代方案分析（用户不用 TT 时）

## 1. 目的

回答：**「用户为什么不继续用 OTA / 找地陪私域 / 现金？」** —— 比只列「竞品公司」更接近真实决策。

## 2. 替代方案清单（扩展表）

| 替代方案 | 用户动机 | 摩擦与风险 | TT 的对应能力（须诚实） |
|----------|----------|------------|-------------------------|
| **OTA 聚合** | 品牌、退改条款、比价 | 平台抽成、体验标准化 | |
| **目的地向导社 / 车队** | 熟人、语言、打包 | 透明度、跨境支付 | |
| **私域成交（微信/WhatsApp）** | 低价、灵活 | **资金与纠纷无中立托管** | **Escrow** |
| **现金 / 当地转账** | 简单 |  chargeback、取证难 | **链上事件** |

## 3. 产品含义

- 若 TT 在某一市场 **未显著优于** 某替代方案，则 **不应强行投放**；见 **`10-growth-engine`** 城市选择。

## 4. **与 UX 验证的关系（替代方案 = 用户真实选项）**

**私域 / 现金** 行的「摩擦」须用 **`14/00` §4** 的 **用户语言** 复述一遍；**禁止** 仅用内部术语写「我们更好」。
