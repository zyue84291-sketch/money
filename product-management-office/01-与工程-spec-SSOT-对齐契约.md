# 01 — 与工程 `spec/` SSOT 对齐契约

**性质**：产品管理与工程之间的 **硬契约**（违反即视为门禁失败风险）。

**PRD / 需求单**：提交评审前须完成 **附录 A（R1～R6）** 自检；**未勾选 R1～R6** 视为 **材料不完整**，评审可拒收。  
**新需求强制入口**：凡进入 **需求评审会** 的材料，须含 **`_registry/新需求-最小入口-模板.md`** 之 **复制区**（「§ 新需求最小入口」）；**缺本节** 与 **缺 R1～R6** **同等** —— **当场拒收**，**不讨论方案**（会议动作见 **`08/meetings/01` §1.2**）。

**产品边界（团队必须统一口径的三句话）**：

1. **不定义技术真值** —— 那是 **`spec/`**。  
2. **不绕过资金语义** —— 以 **`spec/`** 与 **链上** 为准；Office 只作 **对齐叙述**。  
3. **不单模块拍板** —— 跨影响必须 **`02` Impact Map** + **`21`** 决策出口。  

**除此之外**，产品可以在 Office 内 **充分展开** 假设、叙事、实验与运营设计。

## 1. 单一事实来源（SSOT）优先级

1. **资金、订单状态机、链上事件驱动**：以 **`spec/01-总库总览.md`**、**`spec/03-业务流程与风控.md`**、**`spec/10-系统级不变量白皮书-v1.0.md`** 为准。  
2. **HTTP 路由与请求体**：以 **`spec/04-后端与API.md` §3.4** 为准。  
3. **合约、ABI、前后端对齐**：以 **`spec/14-合约-API-ABI-前后端对齐.md`** + `contracts/abi` 为准。  
4. **UI 页面与 RBAC**：以 **`spec/13-1-UI产品级SSOT与页面规范.md`**、**`spec/13-协议级UI设计宪法.md`** 为准。  
5. **对外数字与治理叙事**：以 **`spec/82`**、**`spec/83`**、**`spec/84`**、**`spec/08-4-对外口径包.md`** 及 **`spec/governance-token/`** 为准；**联动脚本**：`scripts/check-governance-doc-linkage`（见 `82` §四）。

**产品文档**（含本 Office）若与上述冲突：**以 `spec/` 为准**，本 Office 须修订并登记 `_registry/`。

### 2.4 临时决策机制（灰度规则）

当 **`spec/` 尚未覆盖** 某能力，但 **业务必须推进**（试点、灰度开关、合同窗口等）时，**唯一合法路径**如下；**否则**按 **§2.5「未更新 `spec` 却已上线」** 或 **违规上线** 处置。

1. **PRD / REQ** 须含 **`临时假设（Gray）`** 小节：写清 **假设内容、证伪条件、Owner、回滚/开关策略**；**不得**与 **`spec/01`** 等 **已定稿不变量** 冲突。  
2. **`_registry/`** 须登记 **一行**（可复制 **`版本登记与责任人模板`**）：**REQ-ID、上线窗口、承诺补写 `spec` 的 Owner + 目标日期**。  
3. **上线后 48h 内**（自然日）须在 **`spec/`** 完成 **对应增补或修订**，并在 PRD / **`_registry`** 留 **回链**；**超时未完成** → 视为 **违规上线**，**记 `违规记录.md`**，并 **冻结** 该能力对外扩大直至补齐。

### 2.5 违规处理原则（Violation Cost System）

> **规则 ≠ 约束**；**有后果**才是约束。下列为 **最低纪律**；与 **`08/meetings/01-Meeting-OS-规则与会议绑定.md`**（**哪场会卡哪条**）、**`_registry/权限边界-强执行.md`**（**谁有权改什么**）同读。

| 违规情形 | 后果（必须执行） |
|----------|------------------|
| **未通过 R1～R6** 的需求 / PRD | **不进入**开发排期（**`17`** / Sprint **不得**收录）；已误收录 → **移出**并 **`_registry`** 记一笔 |
| **未写** **`02`** 三元组（输入自 / 输出至 / Impact Map）的跨模块材料 | **没写 = 不允许完成方案评审**（**不得**进入「通过」结论；**不得**「先通过后补」；纯错别字类除外，见 **`02` §1**） |
| **未更新 `spec` 却已上线**（路由/合约/资金语义与 SSOT 不一致） | 视为 **无效上线**：**回滚** 或 **冻结对外宣称** + **补审**（工程负责人按 **`spec/15` 附录〇** / **`go-live-checklist`** 补齐）；产品侧 **不得**用 Office 文档顶替工程真值 |
| **经营数据 / 对外口径** 与 **`spec/`** 或 **`08-4`** 不一致 | **以 `spec/`（及已定稿 `08-4`）为准**；**Office 必须回滚** 叙述并登记 **`_registry`** |

### 2.5.1 违规执行机制（反滥用）

| 要素 | 约定 |
|------|------|
| **执行人** | **会议主持人**；未指定时由 **PM 负责人** 执行 **拒收 / 不通过** 宣告 |
| **执行时间** | **评审会当场**（需求评审 / 方案评审 / 上线前评审等适用 **`01` §2.5** 的场次） |
| **记录** | 一律写入 **`_registry/违规记录.md`**（含日期、违规类型、处置、执行人、纪要或工单链接）；**会后 24h 内** 完成 |

## 附录 A — 产品需求自检清单（R1～R6）

> 所有 PRD / 需求单在提交评审前必须自检；**未勾选 R1～R6** 视为 **无效需求**（可拒收）。  
> **`spec/` 尚未覆盖** 时 **不得**「空白勾过 R1」：须 **先** 走 **§2.4**（**Gray** + **`_registry`** + **48h** 补 `spec`）；**否则** 按 **§2.5** 违规处置。

### R1 — 是否绑定 spec（SSOT 对齐）

- [ ] 是否标注 `spec/` **文档路径 + 章节/表**  
- [ ] 是否确认 **没有**与 spec 冲突的叙述  
- [ ] 是否排查 **「口头约定但未写入 spec」**  

❌ **常见错误**：写「系统支持 XXX」但 spec 无据；用 PPT/脑图 **替代** SSOT。

---

### R2 — 是否定义状态机 / 链上行为

- [ ] 是否明确 **订单状态迁移**（from → to）  
- [ ] 是否标注 **是否触发链上调用**  
- [ ] 是否区分 **用户签名** vs **系统/执行器执行**  

❌ **常见错误**：把「确认完成」当「放款」；忽略链上 **延迟 / 失败 / 回滚** 分支。

---

### R3 — 是否符合资金语义（强约束）

- [ ] 「Paid / 已支付」是否 **仅**与 **`deposit()`** 等 spec 定义一致  
- [ ] 是否 **误用**「已放款」等 spec 未支持的表述  
- [ ] 是否区分 **Escrow**、**平台收入**、**质押** 语义  

🔴 **一票否决**：任何 **资金语义错误** → 需求 **直接打回**。

---

### R4 — 是否区分三类资金结构

- [ ] **用户资金**（如订单 USDC Escrow）  
- [ ] **身份/角色质押**（staking，见 `spec/81` 等）  
- [ ] **治理/分配**（tokenomics / FeeRouter 叙事，见 `spec/82`～`84`）  

❌ **常见错误**：把 token 当收入；把质押当「用户支付」。

---

### R5 — 是否可发布（门禁映射）

- [ ] 是否映射到 **`spec/15` 附录〇** 或 **`go-live-checklist.md`** 等门禁项  
- [ ] 是否有 **可测** 验收标准（Given/When/Then 或等价）  
- [ ] 是否规划 **evidence**（截图、埋点、试点数据）路径  

❌ **常见错误**：写「功能完成」但 **无**验证标准；**无法**进入发版流程。

---

### R6 — 是否符合产品操作系统

- [ ] 是否符合 **`21-product-operating-system`** 当前 **决策栈**（当季书面版）  
- [ ] 是否通过 **优先级系统**（非临时拍脑袋）  
- [ ] 是否评估对 **North Star（`22-executive-dashboard/04`）** 的影响  
- [ ] 是否能在 **`21/05`** 中填写 **Impact、NS_Weight、Cost、RiskCoef**（或与 RICE **互证**）

👉 **未通过 R6**：**不允许进入开发排期**。
