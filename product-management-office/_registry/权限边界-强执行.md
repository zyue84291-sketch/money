# 权限边界（强执行）

**用途**：把 **「PM 不做什么」** 从叙述变成 **可审计的允许矩阵** —— **谁有权改什么**；与 **`01-与工程-spec-SSOT-对齐契约.md`**、**`21/03-PM决策权边界-RACI.md`** 一致。**冲突时以 `spec/` 与法务书面意见为准。**

**更新**：角色名或审批链变更时，须 **本文件修订 + `版本登记与责任人模板` 留痕**。

---

## 1. 行为 × 允许角色

| 行为 | 允许发起 / 定稿角色 | 备注 |
|------|---------------------|------|
| **修改 `spec/` 技术 SSOT**（含 API、合约、资金语义） | **工程负责人**（或仓库 **CODEOWNERS** 等价角色） | PM **可提需求**，**不得**私改 `spec/` 正文 |
| **修改 PRD / Office 需求叙述** | **产品（PM）**；跨模块须过 **`02`** 评审 | 须满足 **R1～R6** |
| **修改商业模型假设**（take rate 叙事、GMV 口径、单位经济表） | **产品负责人 + CEO（或授权 FP&A）** 双签 | 定稿对外数字仍须 **`spec/08-4`** 一致 |
| **修改资金逻辑 / Escrow 状态机 / 链上参数** | **禁止**由 PM / 运营 **直接改 Office 当真理** | **必须**走 **`spec/`** 变更 + 工程评审 + 门禁 |
| **修改对外价格 / 抽佣策略（用户可见）** | **产品负责人 + CEO**；法务 **高金额 / 辖区** 必签 | 与 **`09`**、**`spec/01`** 对拍 |
| **发布 / 上线宣称** | **工程（发版责任人）+ PM + `go-live-checklist` 勾选** | 未过门禁的上线 → **`01` §2.5** 无效上线处理 |
| **修改运营话术 / 工单 SOP** | **Ops Owner**；涉资金 **必须与 `spec/01` 逐字对齐** | 与 **`19`**、**`Runbook`** 一致 |

---

## 2. 常见越权（显式禁止）

| 越权 | 处理 |
|------|------|
| **PM 直接改 API 契约** | **拒收**；回到 **`spec/04` §3.4** 由工程改 |
| **工程改产品优先级 / 商业承诺** | **回滚** PR 描述；优先级只认 **`21`** + **`12/05`** |
| **运营口头改价或「先上线再补 spec」** | 按 **`01` §2.5** 无效上线 / 回滚叙述 |
