# 规则绑定与执行责任（组织绑定）

**用途**：把 **规则** 绑到 **具体人** —— **未执行 = 责任人承担**（绩效面谈、季审 **`21`**、或升级 **CEO** 按公司制度）；与 **`08/meetings/01`**、**`违规记录.md`**、**`权限边界-强执行.md`** 一致。

---

## 1. 默认绑定（可书面改一次/季）

| 规则 / 门禁 | 第一责任人（执行） | 未执行时 |
|-------------|-------------------|----------|
| **R1～R6**（需求评审） | **需求评审会负责人**（无则 **PM Owner**） | **不得**合并 PRD；**记 `违规记录.md`**；**PM Owner** 须在周会复述整改 |
| **需求评审 §1.2 三步**（最小入口 / R1～R6 / Impact+`21`） | **会议主持人** + **产品负责人**（**拒收权**） | **缺一步仍讨论方案** → **主持人 + 产品负责人** 各记 **`违规记录.md`** 一行（**门禁未执行**） |
| **Impact Map + L 等级**（方案评审） | **方案评审会议主持人** | 会议无效；**主持人** 承担 **补会成本**；重复 → **`21`** 点名 |
| **`违规记录.md` 及时性** | **PM Owner**（可委派 **Scrum Master**，**Owner 仍兜底**） | **24h 未记** 视同 **门禁未跑**；**`21`** 季审单列 |
| **每日执行清单** | **PM Owner** 或轮值 | 漏勾 **连续 3 日** → **`22`** 周会红项 |
| **`spec` 灰度 48h 补写** | **PRD 中 Gray 小节 Owner** + **工程接口人** 双签跟踪 | 超时 → **§2.5 违规上线** 路径 |
| **Office System Health 周检**（**[`Office运行健康度-System-Health-周检.md`](Office运行健康度-System-Health-周检.md)**） | **PM Owner** | **漏填一周** → **`22`** 周会 **红项**；**红级** 未在 **48h** 内开 **`21` 系统修复** → **`21`** 季审单列 |
| **决策复盘（`21/07`）** | **PM Owner** 或 **`21/03` RACI** 仲裁角色 | **漏做一月**（未抽 3 条或未归档）→ **`21`** 季审追问；**连续两月** 同 **偏差标签** → 触发 **`21/01` 栈** 复查 |

**「承担」** 不等于惩罚个人：指 **复盘可见、改进可跟踪、重复升级**。
