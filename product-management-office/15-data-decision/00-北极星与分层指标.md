# 北极星与分层指标

## 1. 北极星（NSM · **已定稿默认 · 可书面改一次/季**）

**当前 NSM（默认）**：**每周 Escrowed GMV**（`weekly_escrowed_gmv`）—— 与 **资金真值** 最近、与 **`spec/01`** Escrowed 状态 **同源**（**`spec/110`** 投影 + **`09/05`** 对账）。

**备选（加权须 CEO 签字）**：每周 **Completed 订单数**；**Net Revenue**（财务关账后 **T+5** 并入 exec 看板）。

**统一约束**：漏斗、实验、**`07/00`** 字典 **必须** 能回答「本条如何 **↑ 或保护** NSM」；否则 **不得进入** **`15/03`** ICE backlog。

**与 `22`**：**`22/04`** 须存在 **NSM 叶子** 与 **`07/04`** 一致命名。

---

## 2. 指标层级体系（**North Star → L1 → L2 · 写死**）

> **横向堆指标 = 无战略**：任何新指标须 **挂靠下列三层之一**；否则 **不得进周会第一屏**。

| 层级 | **指标（`metric_id`）** | **定义一句** | **周会/日会** |
|------|-------------------------|----------------|---------------|
| **NSM** | **`weekly_escrowed_gmv`** | 自然周窗内 **Escrowed 名义 GMV** 之和（USD 口径 CFO 定） | **周会只看这一数**（exec 第一行） |
| **L1（战略过程）** | **`escrow_conversion_rate`** | `escrowed_users` / `commit_users`（7 日滚动） | **日会必看**；**<60%** 触发 **`04` §2** |
| **L1** | **`dau_traveler`** | 日活旅行者（`07/00` 定义） | 与投放/留存同屏 |
| **L2（战术叶子）** | **`wallet_connect_success_rate`** | `wallet_connected` / `wallet_connect_prompt` | 钱包实验 **EXP-0142** 主诊断 |
| **L2** | **`intent_to_commit_rate`**（别名 **intent_rate**） | `commits` / `intent_sessions` | 撮合 / 信任文案实验挂此叶 |

**因果链（信任 → 钱）**：**`trust_score_proxy_a` ↑** 应伴随 **`escrow_conversion_rate` ↑** → **`weekly_escrowed_gmv` ↑**（**`07/04` §3** 最小闭环）；若 **trust ↑ 而 escrow  flat** → **停止**「信任叙事」类全量，**改**钱包/链上。

## 3. 分层指标（输入/过程/输出/健康 · 与上表兼容）

| 层 | 指标示例 | 用途 |
|----|----------|------|
| **输入** | 渠道花费、投放展示 | CAC |
| **过程** | Visit→注册、注册→下单、下单→Escrowed | 漏斗 |
| **输出** | GMV、Completed、NPS | 北极星与质量 |
| **健康** | 争议率、退款率、索引延迟 | 风险 |

## 4. 与 `spec` 的关系

- **链上真值** 指标须标注 **事件源**（`spec/01`、`spec/110`）。  
- **勿**用 DB 近似替代链上对外披露（除非 `08-4` 允许口径）。

## 5. 与 **Penta-001 / CASE-001** 同窗（DD）

- **NSM `weekly_escrowed_gmv`** 的 **定义窗、币种口径、Completed 边界** 须与 **`16/02` §5**（CFO）及 **`16/04` §2.3** 五方 drill **同一工单** 可追溯；**不得** 在 BI 里私自改分子分母。  
- **健康层**（争议率、资金异常）指标若 **升维** 为北极星旁路 → 须对照 **`16/04` §2.4** 字段（触发 → 系统 → Owner → 结果）。
