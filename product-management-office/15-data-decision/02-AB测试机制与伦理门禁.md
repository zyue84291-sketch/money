# A/B 测试机制与伦理门禁

## 1. 最小流程

| 步骤 | 内容 |
|------|------|
| 1 | 假设 + 主指标 + 护栏指标（争议率、退款率） |
| 2 | 样本量 / 周期（或贝叶斯规则预写） |
| 3 | 分流实现（前端开关 / 后端分桶）与 **`x-request-id`** 可追溯 |
| 4 | 统计结论 + 是否全量 |
| 5 | **回滚** 条件写死 |

## 2. 伦理与合规

- **金融展示实验**（费率、托管说明）：须 **法务** 与 **`spec/08-4`**。  
- **不得**对弱势群体做 **不透明** 的价格歧视（司法风险）。
- **法律背书（`16/05`）**：凡实验 UI / 文案触及 **托管性质、退款保证、辖区限制、Gas 代付** → **Scope 列** 须有对应 Memo **版本号**；**超 Scope** → **禁止上线分流**（与 **`14/03` §3** 一致）。
- **会计（`16/02` §5）**：凡实验可能改变 **「何时算收入 / 用户何时失去控制权」** 的表述 → **CFO 表** 更新前 **默认对照组**。

## 3. 工程

- 若实验需新 API → **`spec/04` §3.4** 先行。

## 4. 实验 → 决策（**统计与保护 · 全司默认 · 写死**）

> **无下列三条 = 结果不可上 CFO / 不可全量**：与 **`07/05`、`15/05`** 互证；**例外**须 **`21` 书面**。

| 规则 | **硬标准** |
|------|------------|
| **最小样本** | 每实验臂 **≥ 1000** **sessions**（若主指标为 **user** 级则 **≥ 1000 users**，**二选一** 在工单写死）；**不足** → **只许「延长观察」不许判赢** |
| **显著性** | 主指标 uplift 判赢须 **p < 0.05**（**双尾** 默认；**单尾** 须在 **`06` 台账** 预先登记） |
| **护栏 / 冻结** | **核心转化**（默认 = **`escrow_conversion_rate`**，可工单改为 **`weekly_escrowed_gmv` 环比**）相对对照 **下降 > 5%**（**7 日滚动** 同窗）→ **立即停止** 分流（**技术回滚** + **数据封账**），**48h 内** 复盘入 **`06`** |
| **全量门槛** | 同时满足：**p < 0.05**、**uplift ≥ 讨论 MDE**（默认 **+5% 相对**）、护栏 **未触发** |

**冻结机制（投放）**：与 **`04` §2** 一致 —— **仅 Paid Social** 可 **0 预算**；**自然量 / SEO 不冻**。
