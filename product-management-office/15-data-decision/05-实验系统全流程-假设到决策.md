# 05 — 实验系统全流程（假设 → 决策）

**用途**：标准化 **「做增长引擎」** 而非零散优化 —— **假设 → 设计 → 上线 → 数据 → 复盘 → 决策**；与 **`03-增长实验框架-ICE-RICE`** 排期、**`02-AB测试`** 伦理、**`14`** 漏斗面一致。

---

## 1. 六阶段（每实验一工单 / 一页纸）

| 阶段 | 产出 | 责任人 |
|------|------|--------|
| **H 假设** | 可证伪陈述 + 成功判据 + 影响 **`22` NS 子指标** 哪一叶 | 提出方 |
| **D 设计** | 分流、样本量、风险、**`21/00` 原则** 过栈 | 数据 + 产品 |
| **S 上线** | Feature flag / 路由、**`spec/04`** 若涉 API | 工程 |
| **M 数据** | 看板切片、与 **`07/00`** 事件同源 | 数据 |
| **R 复盘** | 学结论 / 弃 / 扩写 | 提出方 + 产品负责人 |
| **C 决策** | 入 **`17`**、**`20/02`** 或 **停止**；更新 **`02`** Impact 若跨模块 | 产品负责人 |

---

## 2. 与 **20-product-iteration** 的接口

- 每个实验 **必须** 有 **Sprint 或 Kanban 卡 ID**；复盘结论 **48h 内** 写入 **`20/03`** 或周会纪要。

## 3. 拟真实验 ×2（**纸面跑通 · 与 `07/05` 全文一致**）

| 字段 | **EXP-2026-0142** | **EXP-2026-0143** |
|------|-------------------|-------------------|
| **H** | 减少钱包教育步骤 → **Commit→Escrow** 提升 | Escrow 页 **资金流安全说明** → **Intent→Commit** 提升 |
| **C** | 实验组 **少 1 步** `wallet_explain` | 实验组 **折叠卡片**「资金如何托管」 |
| **主指标** | `escrow_conversion_rate` | `intent_to_commit_rate` + `trust_score_proxy`（**`07/04`**） |
| **ICE** | 8 / 6 / 7 = **21** | 7 / 5 / 6 = **18** |
| **样本量（占位）** | 每臂 ≥800 commit_users | 每臂 ≥1200 intent_sessions |
| **成功 / 回滚** | **> +5%** 全量；**< 0** 回滚 | **+5%** 全量；**< -2%** 立即停（信息过载） |
| **详表** | **`../07-metrics-growth/05-拟真实验全链路纸面推演×2.md`** | 同上 |

**变量拆解（方法）**：**自变量** = UI 步数 / 模块曝光；**因变量** = 漏斗率；**控制变量** = 渠道 mix（冻结投放日除外）、合约版本（**须同链同版本窗**）。

**统计与急停（全司默认）**：**`02-AB测试机制与伦理门禁.md` §4** —— 每臂 **≥1000 sessions**、**p < 0.05**、**核心转化 ↓>5% 立即停止**。
