# Web2 → Web3：无感切换与桥接

## 1. 设计原则（与 `spec/06` 一致）

| 原则 | 说明 |
|------|------|
| **渐进披露** | 未进入支付前不强求连接钱包 |
| **解释每步** | approve 与 deposit 分离说明；Gas 与网络 |
| **可回退** | 支付失败有明确恢复路径；不丢订单上下文 |
| **不托管私钥** | 默认路径；代付若存在须 **法务门禁**（`spec/01` §7） |

## 2. 典型断点与对策（占位）

| 断点 | 用户心理 | 产品对策 |
|------|----------|----------|
| 安装钱包 | 麻烦 | 嵌入式说明、视频、WalletConnect |
| Gas / MATIC | 不懂 | 代付或补贴策略（合规先行） |
| 签错网络 | 怕 | 链切换引导、`GET /meta` 合约网络提示 |

## 3. **合规与背书（`16/05`）— 代付 / 补贴 / 路由**

| **能力** | **须出现在 Legal Memo Scope？** | **未签发时产品行为** |
|----------|-----------------------------------|------------------------|
| **Gas 代付 / 补贴** | **是** | **默认关闭** 或 **仅内部钱包**；**禁止** 上生产文案写「免费」 |
| **跨链桥（若接）** | **是** | **功能旗 off**；Roadmap **不得** 写已支持 |

**监控指标**（进 **`07/00`**）：`wallet_connect_fail_rate`、`approve_abandon_rate`、`tx_error_rate_by_step` —— 与 **`14/01` §3`、`16/04` §2.3** 同窗。

## 与 spec/13-1 对齐检查（PR / 发版前 · **≥3 条**）

1. **钱包连接、链切换、approve/deposit** 各步 **页面 / Partial** 是否在 **`spec/13-1`** 有 **路由与组件状态**；与 **`spec/06`** 渐进披露 **无矛盾**。  
2. **错误态 copy**（RPC、gas、用户拒签）是否使用 **`13-1`** 规定 **Hint 组件**；**禁止**未登记的全屏营销挡错误。  
3. 每步是否可映射到 **`spec/04`** API / 埋点（`wallet_*`、`tx_*` 占位）并回写 **`07/00`**。
