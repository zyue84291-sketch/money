# Traveler 与 Guide 核心路径（体验视角）

## 1. Traveler（简版）

```text
认知（Landing/市场）→ 注册/登录 → 发现向导/下单 → 接受报价/确认条款
  → 连接钱包（若尚未）→ approve/deposit → Escrowed → 履约与链下确认
  → 评分（若适用）→ release → 评价
```

- **状态机真值**：`spec/01` §1。  
- **页面 SSOT**：`spec/13-1`、`spec/29`、`spec/53`。

## 2. Guide / `provider`

```text
注册 → 申请向导/资质与质押 → 可接单 → 抢单/接单 → 履约 → 链下确认配合
  → 收款（链上）→ 信誉累积
```

- **准入**：`spec/64`、`spec/81`、`spec/01` §4。

## 4. **真实用户行为验证（Real UX Validation · 非员工）**

> **否则 UX = 设计，不是产品**。下列为 **最低证据包**（可与 **`99-real-world-validation/`** 同窗归档）。

| **项** | **标准** | **产出物** |
|--------|------------|--------------|
| **样本** | **5～10 人** / 轮；含 **≥3 名 Web2 低加密经验** | 招募标准写入 **`99/00`** 或研究工单 |
| **任务** | 完成 **「发现向导 → Commit → 连接钱包 → approve/deposit → Escrowed」** 或 **等价 staging** | 任务脚本链 **`14/01`** 阶段名 |
| **录屏 + 出声思考** | 全程留存（脱敏） | 存 **`99`** 证据目录或 Research Drive |
| **卡点表** | 必须标注 **步骤**（例：**approve 二次确认**、**RPC 错误**、**Escrow 文案不理解**） | **Issue 列表** → **`20`** 或 **`15`** |
| **量化** | **钱包连接失败率**、**approve 放弃率**、**支付页到 Escrowed 时长 P95** | 与 **`07/00`** `metric_id` 对齐；摘要进 **`09/00` §0** 脚注若上融资 |

**频率**：**每 major 支付/钱包 UX 变更** 前 **必须** 跑一轮；**零轮** → **`11/06` B 类** 不得勾「已验证易用」。
