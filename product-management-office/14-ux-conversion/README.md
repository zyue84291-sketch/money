# 14-ux-conversion — 用户体验与转化（UX & Conversion）

**定位**：系统「对」≠ 用户「会用」。本目录回答 **注册→信任→下单→入金** 的 **体验与转化设计**，尤其 **Web2 用户**。

| 对比 | `spec/13`、`spec/05`、`spec/06` | **本目录（14）** |
|------|-----------------------------------|-------------------|
| 侧重 | **UI 宪法、页面 SSOT、DApp 技术** | **漏斗、Onboarding、信任路径、Web2↔Web3 切换策略（产品语言）** |

**硬约束**：视觉、路由、资金区 **不得** 违反 **`spec/13`、`spec/13-1`、`spec/86`**；本目录输出 **PRD 级体验策略**，落地时回写 **`spec/04`**（若涉 API 文案/步骤）。

**实验与增长**：漏斗相关优化走 **`../15-data-decision/05-实验系统全流程-假设到决策.md`**，禁止「无假设上线」。**`00`～`04`** 每篇含 **「与 spec/13-1 对齐检查」** 3 条（PR/发版前勾）。
