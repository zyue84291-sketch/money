# Onboarding：注册 → 首单（魔法时刻）

## 1. 目标

缩短 **Time-to-first-Escrowed**（首单入金），且 **不牺牲 KYC/合规**。

## 2. 检查清单

| # | 项 |
|---|-----|
| 1 | 注册后 **下一步唯一 CTA** 是否明确（找向导 / 发布需求） |
| 2 | **空状态** 是否有示例内容与种子供给（`10-growth-engine`） |
| 3 | 首单前是否完成 **必要** 钱包教育（可跳过 vs 不可跳过写死） |
| 4 | 首单 **金额建议** 是否降低试错成本（产品策略，非 spec） |

## 3. 与 A/B 实验

- 假设与指标 → **`../15-data-decision/`**；**首 Escrow** 指标 → **`../07-metrics-growth/00`** `time_to_first_escrow_hours`。

## 4. **首单前的「法律与会计」硬检查（防 UX 踩线）**

- **`16/05`**：首单路径涉及的 **辖区、支付方式、托管表述** 须在 Memo **Scope** 内；**超 Scope** → **功能拦截**（与 **`05/00`** 同序）。  
- **`16/02` §5**：首单完成触发的 **收入确认** 叙述与 UI **不得** 领先 CFO 定稿表（**`../09-business-model/README.md`**）。

## 与 spec/13-1 对齐检查（PR / 发版前 · **≥3 条**）

1. **注册后下一步唯一 CTA**、**空状态**、**首单路径** 是否在 **`spec/13-1`** **Onboarding / 首访** 区有 **页面与组件 ID**。  
2. **钱包教育** 可跳过 vs 不可跳过是否与 **`13-1`** 状态及 **`spec/06`** 一致；**不**在 **13-1** 外新增强制步。  
3. **首单金额提示** 是否与 **资金区 `13-1`/`spec/13`** 一致且不误导 **Paid** 定义（真值 **`spec/01`**）。
