# 05 — 现金流与商业闭环（用户 → 再投资）

**用途**：补 **「真实商业循环」** 叙事与检查表 —— **用户付费 → 收入 → 成本 → 利润 → 再投资**；数字真值仍以 **`spec/08-4`、`09` 其余篇、`22`** 为准。

**现金细化**：**GMV→银行可花现金**、回款周期、退款与资金占用、burn 见 **`09/06-Cash-Conversion-Loop现金转化闭环.md`**。

---

## 资金流动架构（Money Flow Architecture · **可对账占位**）

> **CFO 卡的不是「GMV 故事」而是「钱走不走得通」**：须有一张 **分层资金流图**（谁付、进哪、谁控、谁执行释放、平台是否经手用户资金）。**账户真名、司法辖区、资产负债表列报** 以 **财务 + 法务 + `spec/01`** 为准；下表为 **对账用骨架**。

| Step | 叙事（产品语言） | **对应 spec（SSOT）** | **账户 / 载体（占位）** | **是否平台资产负债表「持有」用户资金（占位）** |
|------|------------------|----------------------|---------------------------|------------------------------------------------|
| **1** | Traveler → **支付** → 资金进入 **Escrow（链上/协议）** | **`spec/01` §10**、入账接口若涉及 **`spec/04`** | **Escrow 合约 / 协议内余额**（TBD：合约地址清单） | **通常否**（用户资金在 **协议状态**；**列报口径** CFO 定） |
| **2** | **订单完成 / 释放条件** 满足 → Escrow **释放**（状态机终态） | **`spec/01` §10**、**`spec/14`**（若 Escrow 释放链上实现指向此） | **同一 Escrow 路径** → **可拆分余额** | **释放前** 多为 **受限/用户负债叙事**；**释放动作** **执行器 / 合约** 以 **`spec/01`** |
| **3** | **释放结果**：**平台抽佣（take）** + **向导/供给净得** | **`spec/84` §1.1.1**（可分配费用分母）、**`09/00`** | **平台费用池** vs **Guide 应收路径**（TBD） | **抽佣入账后** 平台 **确认收入口径** 见下文 **Revenue vs Cash** |
| **4** | **平台费用池 → 多国/区域/股东叙事分配** | **`spec/83`、`spec/84`** | **FeeRouter / 国库 / 区域桶**（TBD） | **分配** 不等于 **现金已落袋**；与 **`09/06`** 同读 |

**动作**：融资 / 开多币种前，须能 **用链上事件 + 银行子账户流水** **重放** 上表四步；与 **`16/02`** 责任叙事 **零冲突**。

## 对账路径（Reconciliation · **链上 ↔ 中台 ↔ 财务**）

> **CFO 必问**：链上金额 **≠** 财务报表一行数时，**怎么 reconcile**？本节是 **机制占位**，不是 GMV 定义；**对账真值、科目映射、容忍阈值** 以 **CFO + 工程** 书面 Runbook 为准，且须与 **`spec/01`（订单 / Escrow 状态机）**、**`spec/110`（链上索引 / 事件投影）** **同一语义**。

| 层 | **数据源（占位）** | **对账对象** | **SSOT** |
|----|-------------------|---------------|----------|
| **链上** | Escrow **入账 / 释放 / fee** 等 **事件**（含 tx hash、log index） | **中台**「已确认资金动作」金额与笔数 | **`spec/01` §10**、**`spec/110`**、**`spec/10`** 不变量 |
| **中台** | 订单系统 **order_id**、状态、**应付/应收** 投影 | **财务** Revenue / 递延 / 退款 **分录** | **`spec/04`**（若涉及入账 API）、**`09/README` 会计口径** |
| **财务** | **总账 / Revenue 报表** 行 | **链上 + 中台** 日结 **平衡表** | CFO COA、**`spec/08-4`**（对外口径边界） |

**对账方式（占位）**

| 机制 | 说明 | **Owner（占位）** |
|------|------|-------------------|
| **每日对账** | **T-1** 链上聚合 ↔ 订单 **Paid/Escrow/Completed** 计数与金额；**差异 > 阈值** 自动工单 | **工程（索引/账本）** + **财务（勾稽）** |
| **差异处理流程** | 分级：索引延迟 / 合约边界 / 会计口径 / 欺诈 —— **每类** 有 **Runbook 节点** 与 **升级 SLA** | **On-call** → **财务控制** → **`16/04`** 若涉资损 |
| **发布门禁** | **新合约 / 新链 / 会计政策变更** → **须先** 更新对账映射表 + **影子跑** 再切生产 | **工程 TL + CFO 授权** |

**不可验证 = CFO 不信任**：Data Room 中任何 **「链上-derived」** 指标须能 **一键追溯到** 本节三类源之一；细则见 **`05-go-to-market/01-投资人材料-Data-Room-索引.md`** **§Data Room 验证规则**。

## 现金流闭环（Cash Flow Loop · **财务闭合 / CFO 视图**）

> **CFO 不问「逻辑」问「对账」**：每一笔须能说清 **从谁 → 经什么状态 → 到谁 → 平台拿多少 → 剩余可分配池多少**，且与 **`spec` + 财务账** 为 **同一分录口径** 可对上。下列为 **教学闭环图**；**比例、税、时点** 以 **`spec/01`、`spec/08-4`、`spec/83`、`spec/84`** 与 **CFO 定稿表** 为准，**禁止**把示例数字写进对外材料。

### 1.1 单笔闭环（示例：仅说明「钱转一圈」）

**叙事链**：**Traveler** 支付 → **Escrow**（链上/协议状态机，**`spec/01` §10**）→ **完成 / 释放条件满足** → **分账**（平台费、向导/供给、区域等 —— **`spec/83`、`spec/84`**）。

| 环节 | 谁 | 金额（示例 **非真值**） | 对账锚点 |
|------|-----|-------------------------|----------|
| **用户支付** | Traveler → Escrow | **100 USDC** 进入托管合约/托管账户 | **`spec/01`** 状态、`spec/04` 若涉及入账接口 |
| **平台抽佣** | Escrow → 平台可分配费用池 | **10 USDC**（示例 take） | **`spec/84` §1.1.1** 可分配费用分母、**`09/00`** 与 **`spec/08-4`** |
| **向导/供给净得** | Escrow → Guide | **90 USDC**（示例剩余） | **`spec/83`** 区域/角色拆分叙事、**财务** 应付科目 |
| **多国/区域分配** | 平台池内再分 | **X USDC**（占位；**83/84** 表体） | **`spec/83`、`spec/84`** + **FP&A 分摊表** |

**闭合条件（融资/审计必答）**：

1. **链上事件 / 银行流水** 能否 **重放** 上表每一行？  
2. **「Paid / 完成 / 放款」** 与 **`spec/01`** 是否 **零歧义**？（产品话术 **不得** 抢先于状态机）  
3. **收入确认时点** 与 **`09/README`「会计口径说明」** 是否一致？

**与 `09/06` 关系**：本篇画 **「一圈叙事」**；**`06`** 管 **GMV→现金→在途→burn** 的 **营运资本旋钮**。

## 财务闭合（Financial View · **三张表的影子 · 占位**）

> **产品视角 ≠ 公司怎么活**：CFO 需要 **P&L / Cash Flow / Balance Sheet** 的 **粗轮廓**（非精确报表），回答 **赚现金、烧现金、还是锁资金**。下表 **须与 CFO + FP&A** 填实；**数字** 以 **总账 / `spec/01` 事件** 为准。

**A. 三个必答（占位）**

| 问题 | 占位（定稿删除 TBD） | SSOT |
|------|----------------------|------|
| **收入确认时点** | Completed？Escrow **释放**？**分段**确认？ | **`09/README` 会计口径** + **`spec/01`** |
| **成本结构** | 补贴 / BD / 合规 / 技术（云、人）—— **占收入比** TBD；**细桶** 见本篇 **§成本结构（Cost Stack）** | **`12`**、**`09/00`**、财务 COA |
| **资金占用（Escrow 周期）** | 平均 **在途天数**、**受限现金** 是否单列 | **`09/06`**、**`spec/01`**、银行/链上对账单 |

**B. 输出物（融资 / 董事会用）**

1. **粗略 P&L 结构（非精确）**  
   - **收入**：Take rate / 增值（与 **`spec/08-4`** 同源）  
   - **毛利**：收入 − **直接变动成本**（支付通道、争议准备金 **TBD**）  
   - **费用桶**：研发 / 销售与增长 / G&A / **合规与法务**（与 **`12/04`** 池对齐）  
   - **EBITDA / 现金利润**：**TBD**（会计政策由 CFO 定）

2. **现金流方向（正 / 负 / 锁）**  
   - **经营现金流**：订单现金流入 − 退款 − **补贴净流出** − Opex 付现 → **正 / 负** **TBD**  
   - **「锁」**：Escrow / 保证金 / 预付 **不可自由支配** 余额 **TBD**（**BS 影子**：受限现金 / 用户负债 **二选一叙事** 须 **CFO 定**）

3. **与单笔闭环的关系**：**§现金流闭环** 回答 **「一笔单」**；本节回答 **「公司表上长什么样」** —— **同一分录口径** 必须 **互算得通**。

## Revenue vs Cash（占位 · **收入 ≠ 现金**）

> **CFO 必问**：钱进来了，**什么时候算收入**？产品上的 **Paid / Escrow / Completed** 与 **总账收入确认** 可能 **不同日**。**最终以 CFO 定稿 + `09/README` 会计口径** 为准。

| 事件（产品侧） | **现金（Cash）叙事** | **收入（Revenue / P&L）叙事** | **常见差异科目** |
|----------------|----------------------|-------------------------------|------------------|
| **用户支付成功** | **现金（或稳定币）进入 Escrow 路径** —— 可体现为 **经营现金流入** 或 **受限现金**（口径 TBD） | **通常尚未** 全额确认 **平台收入**（尤其存在 **履约未完成**） | **递延收入 / 合同负债（Deferred Revenue）** TBD |
| **订单完成 / 释放条件满足** | **Escrow 释放** 可能 **同时** 触发 **向 Guide 与平台池的拆分** | **收入确认时点** 常见锚点之一（**未必唯一**；分段确认 TBD） | **收入与递延的结转** TBD |
| **退款 / 争议结案** | **现金流出** 或 **Escrow 回退** | **冲减收入 / 计提准备**（政策 TBD） | **准备金、冲回** |

**输出**：任何 **`22`** 或融资表中的 **Revenue** 与 **Cash from operations** 须 **附注** 说明 **与 `spec/01` 状态** 的 **映射一行表**（财务维护）。

## 成本结构（Cost Stack · **公司模型占位**）

> **VC / CFO**：你 **赚钱**，但 **钱花在哪**？本节把 **Opex / 变动成本桶** 摆出来，与 **`09/02` LTV/CAC** **同一世界观**（**CAC 从哪几个桶来**、**边际成本是否吃掉毛利**）。**金额与占比** 以 **FP&A + CFO** 为准。

| 成本桶 | 说明（占位） | 与商业层对齐 |
|--------|--------------|----------------|
| **用户补贴（增长）** | 券、首单、区域试点 —— **须进实验台账 `15`** | **`09/02`** CAC、**`09/06`** 现金烧 |
| **向导激励（供给）** | 签约奖、成单奖、保证金减免（若做） | **`09/03`**、**`spec/81`** 若涉及质押 |
| **支付通道费用** | 法币 / 链上路由、FX、gas（若平台承担） | **`spec/01` / `spec/08-4`**、毛利行 |
| **合规成本** | KYC/AML 供应商、法务 retainer、审计、牌照咨询 | **`12/04`**、**`16`** |
| **技术成本（infra）** | 云、RPC、监控、数据仓库 | **`12/02`**、**`06-delivery-release`** |

**原则**：对外 **Unit economics** 幻灯片 **必须** 含 **「收入 − 直接变动成本 − 增量补贴」** 再讲 **LTV**；否则视为 **收入模型** 而非 **公司模型**。

## 财务与风险更新节奏（Financial Rhythm · **Owner + 频率**）

> **没有节奏 = 系统不会长期运行**：LTV/CAC、Risk、GMV/Revenue **写了不等于会更新**。下列为 **建议默认**；**真 Owner 姓名** 写入 **`22/05-KPI-Owner-Accountability`** 与 **`12`** 预算表。

| 指标域 | **更新频率（建议）** | **Owner（角色）** | **写入会议 / 看板** |
|--------|----------------------|-------------------|---------------------|
| **LTV / CAC / Unit economics** | **月度**（关账后 **5 个工作日内**） | **Finance / FP&A** | **`22/03` 月复盘** 附件 + **`09/02`** 占位表刷新 |
| **Risk 指标**（Exposure、欺诈率、争议率、辖区红灯） | **周度** | **Risk Ops**（或 **`19` + 合规** 联席） | **`22/03` 周会** **红项栏**、**`22/02`** 监控切片 |
| **GMV / Revenue / 有效 GMV**（与 **`spec/01`** 一致定义） | **日更**（T+1 亦可，须书面） | **Data**（BI）+ **Finance 复核** | **`22/04`** 叶子、**`22/06`** 周会 Step2 |

**机制**：上述节奏 **必须** 在 **`21`** 资源与 **`22/03` §0.2**（周会执行纸）中 **可见**；**跳期两次** → 升级 **`21/06`** 或 **`21/07`**（健康度复盘）。

## 1. 闭环七步（每季 exec 勾一次）

1. **获客**（CAC 发生）  
2. **订单与 Escrow**（GMV / 有效 GMV）  
3. **确认与放款**（与 **`spec/01`** 一致）  
4. **收入确认**（抽佣、增值 —— **`09/00`**）  
5. **变动成本**（支付、云、客服边际）  
6. **固定与半固定**（人、办公室 —— **`12/02`**）  
7. **再投资分配**（增长池 / 产品池 / 区域池 —— **`12/04`** + **`21/04`**）
