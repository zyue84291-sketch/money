# 07 — Revenue Truth Layer（财务真值层 · 链上 Escrow）

**用途**：把 **GMV / NSM** 与 **可融资、可入账、可分红** 的 **财务真值** 用 **一条链** 写清 —— 尤其 **链上 + Escrow**：**Escrow ≠ 收入**，**Release ≠ 利润**。否则 **财务不可信**，VC/CFO **直接否决**。

**与 `07/06` 分工**：**`07-metrics-growth/06-财务对账闭环-链上GMV与收入口径.md`** = **对账节奏与误差带**；**本文** = **语义与会计链路（真值层定义）**；二者 **须同版本 bump**。

---

## 1. 真值链（从左到右 · 不可逆混名）

```text
链上/订单事件（spec/01）
    → 产品 GMV / 漏斗指标（07/00）        ← 「发生量」为主
    → 收入确认 / 递延 / 合同负债（09/01 + CFO 表） ← 会计政策
    → 毛利 / 贡献利润（09/02、07/09）      ← 成本匹配后
    → 可分配池 / 协议分账（spec/83/84）   ← 仅在「可分配费用」已定义且合规
```

| 节点 | **常见错误** | **正确原则** |
|------|--------------|----------------|
| **Escrow 入账** | 把 **用户预付** 当 **公司收入** | **多为负债 / 待履约**；收入确认时点 **CFO 书面**（见 **`09/README`** Accounting 表） |
| **Release** | 把 **放款给向导** 当 **公司利润** | **Release** = **履约结算事件**；**利润** = **平台费 − 变动成本 − 分摊** 后 |
| **链上 GMV** | 与 **财务收入** 同一 PPT 数字 | **禁止**；须 **双列** + **`07/06`** 调节脚注 |

## 3. 融资与对外材料的 **最低句**

- 任何 **「收入」** 字样 → **必须** 指明 **GAAP / non-GAAP** 与 **确认时点**；否则 **法务可拦**。  
- **Deck** 前 **48h** → **`11/06`** + **`audit/01`** §2 **无红**。
