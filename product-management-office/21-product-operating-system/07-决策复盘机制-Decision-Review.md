# 07 — 决策复盘机制（Decision Review）

**用途**：**`21`** 里 **决策越来越多**，若 **无人回头看假设与结果**，系统会 **慢慢失真**（**优先级漂移、叙事与真值脱节**）。本文规定 **最低频** 的 **决策复盘**，与 **`22`**（结果指标）、**`15`**（实验假设）互补：**本文盯「拍过的板」对不对**。

**Owner**：**PM Owner** 或 **`21/03` RACI** 中的 **仲裁角色**；**频率**：**每月至少 1 次**（可与 **`22/03` 月会** 合并 **30 min** 专节）。

**联动**：**[`05-决策引擎-Priority-Score-量化模型与系统驱动.md`](05-决策引擎-Priority-Score-量化模型与系统驱动.md)**（决策输入）、**[`../15-data-decision/06-实验台账与复盘模板.md`](../15-data-decision/06-实验台账与复盘模板.md)**（实验层复盘）、**[`../22-executive-dashboard/03-周月经营复盘与董事会材料.md`](../22-executive-dashboard/03-周月经营复盘与董事会材料.md)**（经营层）。

---

## 1. 每月固定动作

1. **抽取 3 个关键决策**（上月或当季仍生效的）：优先 **上过 `21` 联席 / 改过栈 / 动过预算 / L3+ Impact** 的条目；**写清 REQ-ID 或 决议链接**。  
2. **逐条对比**（一页纸 / 表格三列）：  
   - **当时假设**（含数据、时间窗、依赖 **`spec` 段落**）  
   - **实际结果**（**`22`** 指标 + **现金三问** 见 **`09/README`**、**`09/06`**）  
   - **是否继续**（Continue / Tweak / Kill）与 **Owner + 日期**  
3. **归档**：放入 **`_registry/`** 一行摘要（或你们 Wiki 固定页）并 **链到月纪要**。

---

## 2. 输出物（最低限度）

| 产出 | 说明 |
|------|------|
| **决策正确率（滚动）** | 定义：**Continue** 或 **Tweak 后达标** = 当次记 **1**；**Kill 且证实早该杀** = **1**；**Continue 但核心假设被证伪** = **0**。按 **近 12 次** 复盘滚动 **%**（样本小则标 **「低置信」**）。 |
| **决策偏差类型** | 每月 **打标签**（可多选）：**过度乐观**、**漏算 Impact 链**、**spec 假设错**、**外部变量**、**执行 slippage**、**指标口径漂移** 等；**连续两月** 同标签 **≥2** → 触发 **`21/01` 栈** 或 **`12/05`** 资源复查。 |

---

## 3. 与实验复盘的关系

- **实验复盘（`15`）**：**假设 → 数据 → 下一实验**。  
- **决策复盘（本文）**：**已拍板 → 假设 vs 世界 → 是否还值得占资源**。  

二者 **都** 要做；**不得**用实验 busywork **替代** 对 **战略/资源级决策** 的回头检。

---

## 4. 决策验证机制（强制 · 每个 **`21`** 决议）

> **「做了决定」≠「做了正确决定」** —— 与 **`21/05` §0**（当下绑定指标）衔接：决议 **落地时** 必须 **同时** 建 **验证票**。

**每个 `21` 书面决议**（含周会 **`22/06`** 第四节产出）须包含：

1. **标记对应指标（`22`）**：**`22/04`** 叶子名 + **当前值**（或链 BI）。  
2. **设置验证窗口**：**7 天** 或 **14 天**（**L3+ / 资金相关** 默认 **14**）；**起算日** = 决议日或上线日（**书面写死**）。  
3. **到期必须复盘**（可并进 **`22/03` 周会** 固定 5 min 栏）：**是否达预期** → **Continue / Tweak / Stop / Rollback**；**未复盘** → 视同 **`21/05` §0** 未完成闭环，**下一张同类决议** 冻结至补齐。

**归档**：复盘结论 **链回** 原决议（纪要 / **`20/03`** 升级项 / **`15/06`** 一行）。

---

## 5. 决策责任绑定（强制 · **不是追责、是纠偏**）

**每个 `21` 书面决议**（含 **`21/01` §4** 资源块）须 **显式** 绑定：

| 角色 | 绑定对象 |
|------|----------|
| **指标 Owner** | **`22/04`** §2 或 §2.1 对应行的 **主 owner**（**对指标定义与阈值解释负责**） |
| **决策 Owner** | **`21/03` RACI** 中的 **仲裁 / 产品负责人** 或纪要中的 **Decision Owner** 字段（**唯一**） |
| **执行 Owner** | 同 **`21/01` §4** **执行 Owner**（**对交付节奏负责**） |

**当 `§4` 验证失败**（未达预期、或 **Stop/Rollback**）：

1. **必须复盘**：**7 日内** 写入 **`20-product-iteration/03`**（或当周 **`meetings/`**）**一条**「决策-结果偏差」记录，含 **事实 / 假设错在哪 / 下一动作**。  
2. **标记责任链**：并列 **决策 Owner + 指标 Owner + 执行 Owner** 三方 **各自纠偏项**（例：假设错 / 口径错 / 执行 slippage）；**禁止**只写「团队反思」无主名。  
3. **连续两次** 同一决策链 **验证失败** → **升级 `21/02` L3** 并 **冻结** 同类决议至 **书面补丁**。
