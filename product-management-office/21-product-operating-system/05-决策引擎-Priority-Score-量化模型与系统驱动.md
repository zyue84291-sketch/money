# 05 — 决策引擎（Decision Engine）：Priority Score 与系统驱动

**用途**：回答 **「为什么这个需求优先 / 为什么这个实验被放大 / 为什么这个不做」** —— **用可复核的算式与权重**，避免退化为 **「因为 21 这么说」** 的 **权威驱动**；与 **`04`**（RICE/截断）、**`22/04`**（NS 权重）、**`01`** 附录 **R6** 对齐。

**性质**：**`21`** 子系统的 **决策算法 SSOT**；**不**替代工程 **`spec/`**。

**真实运行（非装饰品）**：每个 **Sprint / 双周规划** 须在 **`10-决策引擎运行记录-PriorityScore-RUN-001.md`**（及后续 RUN-00x）**新增一行或一节** —— **含 Top5 + Kill≥1**；**全空** → **不得宣称 Decision Engine 已用于排期**。

---

## 0. 决策必须绑定指标（强制 · **`21` ↔ `22` 硬绑定**）

> **决策 ≠ 拍脑袋**：凡记入 **`21`** 联席 / 仲裁 / 资源板（**`12/05`**）的 **书面决议**，**必须**同时满足下表（**含指标 + 资源**，与 **`21/01` §4** 一致）；**缺一** → **无效决策**（**不得**进入 **`17`** 排期、**不得**作为 **`15`** 放大依据），须在 **7 日内** 补全或 **宣告作废**。

| 必填项 | 要求 |
|--------|------|
| **影响的 NS 子指标** | **逐条** 指向 **`22/04`** §2（或 §2.1 明细行）中的 **具体叶子名**；**禁止**只写「提升体验」 |
| **预期变化** | 对每个叶子写 **↑ / ↓** 及 **幅度区间**（例：首单转化率 **↑2～4pt**） |
| **验证方式** | **二选一或组合**：**`15`** 实验（写清 **REQ/台账 ID**）或 **`22`** 观察（写清 **周会看板字段 + 观察窗**） |
| **执行 Owner（唯一）** | 同 **`21/01` §4**；与 **PriorityScore** 的 **Cost** 责任人 **可同一人**，但 **必须写名** |
| **投入资源** | **人天 / Squad**；与 **`12/05`** 板 **对拍**；**0** 须说明理由 |
| **被暂停事项** | 同 **`21/01` §4**；**不得**「加活不减活」 |
| **决策拆解（Task 表）** | 同 **`21/01` §5`**：**≥3 行** + **`Status` 四态** 或 **书面豁免**；**缺表 / 缺 Status 列** → **无效决策** |

**与 `21/07` 关系**：本节解决 **「当下有没有依据」**；**`21/07`** 解决 **「到期后对不对」**（验证窗 + 复盘 + **§5 责任链**）。

## 1. 问题陈述

若仅有 **决策出口（`21`）** 与 **R6（须符合决策栈）** 而无 **可量化比较**，团队仍会听到：

- 「为什么是我被砍 headcount？」
- 「为什么增长实验不能放大？」

**必须**把「拍板」还原为 **同一套输入 → 同一套公式 → 可追溯排序**；**`21/01`** 的栈顺序用于 **同分 tie-break** 与 **原则冲突**，**不**用于跳过量化。

---

## 2. 核心公式（Priority Score）

对任意候选项 **i**（需求、实验包、技术债窗口、运营专项）定义：

\[
\textbf{PriorityScore}(i) = \frac{\textbf{Impact}(i) \times \textbf{NS\_Weight}(i) \times \textbf{Strategy\_Weight}(i)}{\textbf{Cost}(i) \times \textbf{RiskCoef}(i)}
\]

**纯文本**：`PriorityScore = (Impact × NS_Weight × Strategy_Weight) / (Cost × RiskCoef)`。

| 因子 | 取值约定 | 数据来源 |
|------|----------|----------|
| **Impact** | **1～10**（对当季 NS 或约定 KPI 叶的 **可辩护** 影响幅度） | PRD / 实验假设 + **`22/04`** 映射行 |
| **NS_Weight** | **0.2～1.0**（与 **`22/04`** 中该叶权重 **一致**；未映射叶默认 **0.3** 且须说明） | **`22-executive-dashboard/04`** |
| **Strategy_Weight** | **0.5～1.5**（当季 **`21/01`** 书面栈：增长/风控/收入/UX 的 **显式加成**） | **`21/01`** |
| **Cost** | **人天**（或等价 **Engineering Week**；**≥0.5**，避免除零） | **`17/03`**、**`12/05`** 板 |
| **RiskCoef** | **≥1.0**（合规、资金语义、品牌、运维 **风险乘子**；无额外风险取 **1.0**） | **`16`**、**`19`**、**`01`** R3/R4 一票否决项 → 直接 **∞（不可进当期）** |

**排序规则**：在 **Cost、Risk、合规底线** 可接受的前提下，**PriorityScore 高者优先**；同分走 **`21/01`** 默认裁决顺序 + **`21/02`** 升级。

---

## 3. 反模式（显式禁止）

| 反模式 | 纠正 |
|--------|------|
| 「**VP 声音大**」 | 未写入 **Impact/NS_Weight/Cost** 的，**不得**进 **`12/05`** 资源板 |
| 「**上次也这么做**」 | 须重新跑分；**`99`** 或 **`22`** 已证伪的假设 **降权** |
| 「**工程师觉得快**」 | **Cost** 可低，但若 **Impact×NS_Weight** 低，仍不应挤占 P0 |

---

## 4. 与 **`04`**、**R6** 的关系

- **`04-统一优先级框架`**：RICE / ROI / 四象限与 **本 Score** 属 **同一决策家族**；**Epic 级** 可只用 RICE，**须能解释**与 **NS_Weight** 一致。  
- **`01` 附录 R6**：未通过 **决策栈 + North Star 影响陈述** 的，**Score 不予计算**（直接拒收）。
