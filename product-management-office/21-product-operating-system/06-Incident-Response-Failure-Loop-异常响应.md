# 06 — Incident Response & Failure Loop（异常响应系统）

**用途**：当 **正常飞轮**（**`22`→`15`→`21`→执行→`99`**）跟不上 **现实打击** 时，进入 **可宣布、可退出** 的 **紧急模式** —— 避免 **「没人宣布紧急 = 慢性死亡」** 或 **「各团队各自救火 = 二次伤害」**。  
**战术层「做了没效果」**（非事故暴跌）走 **`21/01` §5.1** / **`22/03` §0.5**；**本节** 管 **阈值触发的战情与冻结**。

**性质**：**`21`** 有权 **临时改写当季优先级**；**`19`**、**`16`** 提供 **话术与 Runbook**；**`15`** 在冻结期 **停止新开/放量**（除非 **`21`** 书面豁免）。

---

## 1. 触发条件（示例阈值，须按 **`22/02`** 实际口径校准）

| 级别 | 条件（满足其一可提议进入） | 提议人 |
|------|---------------------------|--------|
| **L2 黄色** | NS 子指标连续 **2 周** 跌出 **黄线**；或单周 **争议工单** ↑ **25%** | **`22`** owner / Ops |
| **L1 红色** | GMV / 核心转化 **↓30%**（同比/环比 **事先定义**）；或 **争议率 ↑50%**；或 **合规突发**（监管函、支付通道冻结） | **CEO 或 `21` 仲裁人** |

**进入 L1**：须 **书面宣布**（Slack/邮件主题 **`[INCIDENT-L1]`** + 文档链接），并 **2h 内** 拉起 **War Room**。

---

## 2. L1 强制动作（Failure Loop）

1. **War Room**（战情室）：固定 **每日** 短站会 + **`22`** 一页纸；**RACI** 见 **`12/01`**、**`21/03`**。  
2. **`15` 冻结**：**暂停**新开实验与 **放量**；在跑实验 **只保留** 与事故归因 **直接相关** 的（**`21`** 单列名单）。  
3. **`21` 临时优先级**：发布 **≤7 天** 有效的 **临时决策栈**（可覆盖当季 **`21/01`** 中的 **非安全底线** 条款）。  
4. **`22` 每日复盘**：红项根因分类（产品 / 运营 / 外部 / 合规）；同步 **`19`**、**`24`** 对外口径 **单出口**。  
5. **`17`/`18`/`14`**：仅执行 **War Room backlog** 内条目；**禁止**侧车需求。

**工程 / 资金真值**：仍以 **`spec/`**、**`Runbook`** 为准；本文件只约束 **产品侧资源与实验纪律**。

---

## 3. 退出条件

| 条件 | 说明 |
|------|------|
| **指标恢复** | 核心 NS 子指标回到 **黄线内** 连续 **N 天/周**（与进入前约定一致） |
| **归因闭环** | 根因、修复、验证证据写入 **`20`** 复盘 + **`_registry`** 一行 |
| **书面解除** | **`21`** 发布 **「退出 L1」**；恢复 **`15`** 常规门禁与 **`21/01`** 当季栈 |
