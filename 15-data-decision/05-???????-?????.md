# 05 — 实验系统全流程（假设 → 决策）

**用途**：标准化 **「做增长引擎」** 而非零散优化 —— **假设 → 设计 → 上线 → 数据 → 复盘 → 决策**；与 **`03-增长实验框架-ICE-RICE`** 排期、**`02-AB测试`** 伦理、**`14`** 漏斗面一致。

---

## 1. 六阶段（每实验一工单 / 一页纸）

| 阶段 | 产出 | 责任人 |
|------|------|--------|
| **H 假设** | 可证伪陈述 + 成功判据 + 影响 **`22` NS 子指标** 哪一叶 | 提出方 |
| **D 设计** | 分流、样本量、风险、**`21/00` 原则** 过栈 | 数据 + 产品 |
| **S 上线** | Feature flag / 路由、**`spec/04`** 若涉 API | 工程 |
| **M 数据** | 看板切片、与 **`07/00`** 事件同源 | 数据 |
| **R 复盘** | 学结论 / 弃 / 扩写 | 提出方 + 产品负责人 |
| **C 决策** | 入 **`17`**、**`20/02`** 或 **停止**；更新 **`02`** Impact 若跨模块 | 产品负责人 |

---

## 2. 与 **20-product-iteration** 的接口

- 每个实验 **必须** 有 **Sprint 或 Kanban 卡 ID**；复盘结论 **48h 内** 写入 **`20/03`** 或周会纪要。
