# 冷启动与城市级模型

## 1. 城市选择框架（占位）

| 评估维 | 权重（和=100） | 说明 |
|--------|----------------|------|
| 入境游规模 / 客单价 | | 可引用公开统计，与 `spec/84` 叙事一致时注明来源 |
| 供给可招募性 | | 向导密度、语言 |
| 支付与合规难度 | | 与法务对齐 |
| Web3 渗透 / 钱包普及 | | 若做 Web3 获客 |

**输出**：**Phase 0 试点城市清单（1～3）** + 每城 **供给目标 / 旅行者目标**。

## 2. 供给先行 vs 需求先行

| 策略 | 适用条件 | 风险 |
|------|----------|------|
| **供给先行** | 体验高度依赖向导质量 | 空列表体验差 → 须「种子向导」运营 |
| **需求先行** | 强品牌或渠道导流 | 供给跟不上 → 退款与差评 |

**TravelTrust**：订单依赖 **Escrow + 向导履约**，默认倾向 **供给种子 + 有限城市聚焦**；写进你的假设表。
