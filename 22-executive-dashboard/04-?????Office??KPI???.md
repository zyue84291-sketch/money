# 04 — 北极星与全 Office 模块 KPI 绑定表

**用途**：解决 **「增长看增长、产品看功能、运营看转化」** 的 **方向分裂** —— **唯一 North Star** 下，各支柱 **必须挂至少一条 KPI 叶子**；与 **`00`**、**`07/00`**、**`15/00`** 同源。

---

## 1. 绑定规则（门禁）

1. **全公司只认一个 NSM**（定义见 **`00-北极星指标与唯一核心.md`**）；变更须 **CEO/产品负责人书面** + 更新 **`22/03`** 周会附件。  
2. **每个支柱目录** 在自家 README 或 Epic 表维护 **「本模块贡献的 NS 子指标」** 一行（可链到本表 **§2**）。  
3. **周会只看**：NS + 3 支撑 + 红项（**`03-周月经营复盘`**）；**禁止**各团队自带一套「自己的北极星」上 exec 会。  
4. **无阈值 = 只有解释、没有决策**：§2.1 **明细表** 每行 **必须** 有 **Threshold** 与 **Status**（与 **`22/02`** 告警阈值 **同源或可链**）。

---

## 2.1 明细行（强制 · **阈值 + 状态**）

> **用途**：把 **`22/04`** 从「绑定表」变成 **周会可直接判红绿** 的纸；与 **`21/00` §4 T2**、**`06` 周会** Step2 对齐。**Status** 每周五或周会当场 **人工更新**（或链 BI 规则）。

| 模块 / 支柱 | 子指标（叶子名） | Owner | **阈值（Threshold）** | **当前状态** |
|-------------|------------------|-------|-------------------------|----------------|
| 例：**14** UX | 首单转化率 | PM | **<20%（周均）→ 异常**；**<15% → Critical** | Warning |
| 例：**22** dashboard | NS 周环比 | CPO | **连续 2 周为负 → 触发 `21`**（同 **`21/00` T1**） | Normal |
| （复制加行） | | | | |

**状态枚举**：**Normal** / **Warning** / **Critical**（**Critical** 默认 **当周必须** 进 **`21` 或 `21/06`** 之一）。
