# 信任设计：评价、担保、UI

## 1. 三层信任（产品语言）

| 层级 | 手段 | `spec` 锚点 |
|------|------|-------------|
| **感知** | 向导 DID、历史单、媒体素材 | `spec/64`、`spec/30` |
| **交易** | Escrow、明确费用与终态、争议入口可见 | `spec/13` 资金区、`spec/01` |
| **事后** | 评价、申诉、客服 SLA | `spec/03` |

## 2. UI 纪律

- **资金区**：银行级克制；**`spec/13`**、**`spec/86`**。  
- **不得**用娱乐化动效掩盖 **金额/地址/网络**（`spec/13-1` 异常态）。

## 3. 首访信任路径（检查清单）

- [ ] 首屏 10 秒内说明 **「钱去哪、谁保管私钥」**  
- [ ] Escrow 页可解释 **「未完成 vs 已入金」**  
- [ ] 争议入口 **不隐藏**（但不误触）
