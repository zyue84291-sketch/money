# 收入结构：佣金 / 手续费 / 增值服务

## 1. 三层收入（产品语言）

| 层级 | 典型形态 | 与 TravelTrust 的锚点 |
|------|----------|------------------------|
| **交易抽成** | 订单 GMV 上的 take rate / platform fee | **`spec/01` §5、§10**（`platformFeeBps`、终态是否计费）；链上 Escrow 释放逻辑 **`spec/14`** |
| **流量与撮合** | 列表曝光、优先排序、广告（若做） | **产品决策**；须单独 PRD + 合规评估 |
| **增值服务** | 保险、加急仲裁、企业套餐、API / SaaS | **产品决策**；不得默认已上线 |

## 2. 与「83 = 分配」的边界

- **`spec/83` / `spec/84`**：描述 **已进入 FeeRouter 叙事的那部分「可分配平台费用」** 在国家桶 / Global 等之间的 **比例分配**。  
- **本文件**：描述 **平台对单笔订单抽多少（BPS）**、是否还有 **订阅 / 上架费** 等 **收入来源设计**；两者相乘才接近「完整 P&L 故事」。

## 3. 工程对齐检查（发版/融资前）

- [ ] `platformFeeBps` 与 **`spec/01` §5 十**、Escrow 表一致  
- [ ] 对外 PPT 中的「平台抽成」数字 **≤ `spec/08-4`** 允许表述  
- [ ] 若涉及 **84** 中「可分配费用 100% 分母」，已与 **`spec/84` §1.1.1** 同批核对（与 gas/仲裁费/slash **正交**）
