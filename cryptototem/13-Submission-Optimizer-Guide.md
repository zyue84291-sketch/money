## 1. 60-second reviewer path

| Phase | Time | Read / verify | Pass criterion |
|-------|------|---------------|----------------|
| **First** | 0–15s | **§1 System overview** (three layers) | Marketplace / Escrow / Governance separated; TTG governance-only |
| **Second** | 15–30s | **§2 Fund flow separation** | User USDC escrow only; TTG not in payment path |
| **Third** | 30–45s | **§3–4 Live / Not live** | Demo YES; escrow + TTG NOT DEPLOYED |
| **Risk** | 45–60s | **§5–6 Risk control + verdict** | No investment/yield/return narrative; auditable links |

## 2. Optimal submission section order

| Order | Section | Why first |
|-------|---------|-----------|
| 1 | §1 System overview | Answers "what is this?" in 15s |
| 2 | §2 Fund flow separation | Highest ICO-listing scrutiny point |
| 3 | §3 What is live | Proves verifiable signals before gaps |
| 4 | §4 What is not live | Deployment honesty |
| 5 | §5 Key risk control | Required; placed after facts |
| 6 | §6 Reviewer verdict target | Confirms low-risk structure intent |
| 7 | Team + footer | Verification last |

**Do not lead with:** roadmap, tokenomics, team bios, or marketing pitch.
