**用途**：定义 **「现实世界验过」** 的最低 bar；未达标结论 **不得** 写进 **`11`** 主叙事或 **`05`** 对外定稿。

---

## 1. 三层验证（建议顺序）

| 层 | 对象 | 最低证据 |
|----|------|------------|
| **L1 可用性** | 小样本 Guide / Traveler | 录屏 + 任务完成率 + **P0 阻断清单**（见 **`01`**） |
| **L2 城市试点** | 1～2 城、可控供给 | **`02`** 周更数据包 + 客服工单样本 |
| **L3 商业验证** | GMV、抽佣后收入、漏斗 | **`03`/`04`** 与 **`07/00`、`22/00`** **同一口径链接** |

## 2. 反馈闭环（必须落地）

1. **发现** → 记入 **`20-product-iteration/00`** 或工单系统。  
2. **分类** → 产品 / 工程 / 运营 / 合规 owner（**`21/03`**）。  
3. **回写** → 更新 **`14`/`18`/`19`/`09`** 中受影响的段落，并在 **`02-系统联动规则与模块Impact-Map.md`** 对应小节 **补一行变更注记**（或 `_registry`）。  
4. **贯穿战略** → 按 **`05-反馈贯穿全Office-flywheel.md`** 进入 **`03`→`20`→`01`**。
